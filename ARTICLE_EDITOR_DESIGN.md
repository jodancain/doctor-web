# 医院宣教知识库 - 云文档编辑方案

## 1. 现状与痛点分析

在当前版本中，文章编辑经历了几次调整：
1. **初期**：模态弹窗（Modal） + 原生多行文本框（Textarea）。门槛极高且极易出错。
2. **中期（失败）**：引入大型富文本库（WangEditor/React-Quill） + 手机实时预览。由于项目使用了最新的 **React 19**，这些相对底层的旧版富文本编辑器存在严重的兼容性问题，导致 React 运行崩溃（页面白屏）。另外，手机实时预览的布局虽然炫酷，但对实际编辑长文体验帮助有限，甚至有些挤占屏幕。

## 2. 优化目标 (云文档化)

根据用户的最新要求：“不用手机实时预览，患者端可以自动适配就行，尽可能简单好用像一般的云文档功能一样就行”。

我们将完全重构这一模块，打造一个**“类似飞书文档/语雀/Notion的沉浸式极简在线文档编辑体验”**。

## 3. 核心交互与界面设计 (UI/UX)

摒弃手机预览和冗杂的侧边栏，将文章发布/编辑重构成**一个极简的居中长文档（Cloud Document Style）**。

### 3.1 极简居中排版
*   **全局背景**：浅灰色的舒适底色（`bg-[#f8f9fa]`）。
*   **白纸体验**：居中一张最大宽度 `max-w-4xl` 的白纸（Card），所有创作都在这张“纸”上完成。
*   **顶部悬浮工具栏 (Sticky Header)**：吸顶显示的返回按钮、草稿/发布按钮。随时保存，不占空间。

### 3.2 沉浸式文档结构
*   **自由封面图**：文档顶部支持直接粘贴 URL 来添加大图作为文章头图，体验类似于 Notion。
*   **大字号标题**：使用极大的字号、无边框的 `<input>`，提示用户“无标题文档”。
*   **轻量化元信息**：分类、阅读时长等不再采用堆砌的表单，而是变成小号文字、下拉框，自然地嵌入在标题下方。
*   **无感富文本区**：去除了所有丑陋的灰色边框，将工具栏与编辑区域自然融合，仿佛在纸上书写。

## 4. 技术栈选型方案

由于 **React 19** 的严格模式及新机制，很多包含遗留 API（如 `findDOMNode` 或对 Refs 过度依赖）的富文本库全部阵亡。

### 4.1 最终选型：`react-simple-wysiwyg`
*   **优势**：0 依赖的纯 React 组件，直接使用浏览器原生的 `document.execCommand` 或 `Selection API` 实现。
*   **兼容性**：完美适配 React 18/19，极其轻量，不会引发任何崩溃或白屏。
*   **功能**：开箱即提供加粗、斜体、列表、插入链接和图片的基础功能，足够医生或科普人员发布排版简洁的宣教内容。

### 4.2 小程序端自适应 (Mini-Program Auto-Adaptation)
富文本生成的内容为标准 HTML 标签（`<p>`、`<b>`、`<img>` 等）。
小程序的 `<rich-text>` 组件将自动接管：
*   我们不再在 Web 端做强行模拟，直接利用小程序 `<rich-text>` 的能力。
*   为了防止图片超出手机屏幕，可以在小程序端的全局或页面的 CSS / 解析中统一添加 `max-width: 100%`。

### 4.3 编辑区布局滚动优化 (Scroll Fixes)
在早期的极简实现中，采用了 `overflow-hidden` 或固定高度的容器导致文档必须在局部滚动，此时编辑器自带的悬浮工具栏 (Toolbar) 要么被遮挡，要么随页面滚走。
优化后我们将整个页面的容器设为自然向下延伸（`overflow: visible`, `height: auto`），移除富文本容器自己的滚动条，使用浏览器的原生滚动条。同时利用 `position: sticky; top: 80px;` 和样式覆盖，使得编辑器的格式工具栏可以变成一个漂亮的**毛玻璃药丸按钮栏**悬浮于页面上方。

## 5. 实施状态与总结

*   [x] **彻底解决白屏**：移除了不兼容的 `react-quill` 和 `@wangeditor` 及其依赖。
*   [x] **引入轻量编辑器**：使用稳定、现代的 `react-simple-wysiwyg`。
*   [x] **UI 极简改造**：摒弃左右分栏，重构成类似云文档的一页纸（Single Page Document）排版模式。
*   [x] **沉浸式滚动体验升级**：彻底解决局部滚动（overflow: hidden）和由于内容过长导致工具栏无法正确吸顶悬浮的问题，让用户滚动整个网页而不是框内内容。工具栏会智能悬浮于页面顶部中央。
*   [x] **交互升级**：支持了更为优雅的封面图挂载、元信息录入和吸顶工具栏体验。
