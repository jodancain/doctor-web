## [v2.0.2] — 2026-03-30 小程序医生端患者访问修复

### 修复 (Fixed)
- **修复小程序医生端“无法访问患者信息/患者列表为空”回归问题**：
  - **根本原因**：
    - `server/services/miniprogram/services/user-service.cjs` 的 `getDoctorPatients`、`getDoctorDashboardStats` 在优化后使用 `where({ role: 'user' })` 精确查询，导致历史患者账号（`role` 缺失/枚举不一致）被漏掉。
  - **解决方案**：
    - `getDoctorPatients` 改为 `role != doctor` 宽口径查询，并在服务端二次过滤（排除医生、仅保留有 `_openid` 的账号）；
    - `getDoctorDashboardStats` 统一为同口径统计，避免仪表盘与列表不一致；
    - `getDoctorPatientOverview` 增加医生账号拦截，防止把医生记录当患者详情返回。

- 更新 `goutcare---doctor-portal (1)/DOCTOR_PORTAL_TROUBLESHOOTING.md`：新增“小程序医生端无法访问患者信息”排障章节。

## [v2.0.1] — 2026-03-30 AI 对话稳定性修复

### 修复 (Fixed)
- **修复医生端 AI 对话“返回 200 但气泡为空”问题**：
  - **根本原因**：
    - 腾讯元器 SSE 响应存在分片，旧版按 chunk 直接切行解析，导致被拆分 JSON 丢失；
    - 返回字段并不总是 `choices[0].delta.content`，部分响应无法被旧逻辑提取。
  - **解决方案**：
    - `server/services/providers/yuanqi.ts` 增加流式缓冲解析，兼容 `delta.content` / `message.content` 等字段，并增强非 200 错误透传；
    - `server/routes/ai.ts` 增加兜底逻辑：流式阶段未产出文本时自动回退一次非流式调用；
    - `pages/AIChat.tsx` 增加前端空响应兜底提示，避免出现空白助手气泡；
    - 当元器返回 `400` 等配置类错误时，前端可看到可读错误提示，不再“无响应”。

### 文档 (Docs)
- 更新 `all/MIGRATION_DEPLOY_GUIDE.md`：新增生产环境 AI 对话排障步骤。
- 更新 `goutcare---doctor-portal (1)/DOCTOR_PORTAL_TROUBLESHOOTING.md`：新增 AI 对话空白问题定位与修复说明。

### 修复 (Fixed)
- **修复问卷详情页语法错误导致上传/编译失败**:
  - **问题现象**: 微信开发者工具上传代码时报错：`Expecting Unicode escape sequence \uXXXX`，指向 `packages/records/questionnaire/detail/index.js` 第 55 行附近。
  - **根本原因**: `onSingleChange` / `onMultipleChange` / `onTextChange` / `onRatingChange` 中 `setData` 的动态字段名被破坏（`answers.${id}` 模板字符串损坏为非法转义字符），触发 Babel 解析失败。
  - **解决方案**: 将上述方法中的 `setData` 改为使用计算属性名（模板字符串键 `answers.${id}`），正确写入 `answers` 各题目字段。

- **修复患者端知识库文章“全部为空”问题**:
  - **问题现象**: 患者端“健康知识库”页面显示“暂无相关文章”，热门推荐卡片无正文，但管理端已存在文章数据。
  - **根本原因**: 患者端列表查询使用了强匹配条件 `status === '已发布'`。历史导入数据中部分文章 `status` 字段缺失或采用了不同值（如 `published` / 布尔值），导致有效文章被全部过滤。
  - **解决方案**:
    - 在 `miniprogram/pages/health-info/index.js` 增加发布状态兼容函数，统一识别多种“已发布”状态，并兼容无 `status` 的历史数据。
    - 将列表查询改为“先取数据再按兼容规则过滤”，避免因为状态值差异导致患者端空列表。
    - 在 `miniprogram/packages/resources/article/index.js` 中同步修复相关推荐逻辑，保证详情页与列表页的可见性规则一致。
    - 在 `miniprogram/pages/health-info/index.wxml` 中为热门推荐增加条件渲染，避免无数据时出现空白卡片。

- **修复云函数依赖导致的功能执行失败问题**:
  - **问题现象**: 在进行任何增加记录操作（如添加饮食记录、用药提醒）时，控制台报错 `Error: Cannot find module 'wx-server-sdk'` 或 `FUNCTIONS_EXECUTE_FAIL`。
  - **根本原因**: `healthDataService` 云函数的 `package.json` 中配置的 `wx-server-sdk` 依赖版本使用了 `~2.6.3` (严格匹配次版本号)，导致在最新云环境（Node.js 16/18）下未能自动拉取兼容的高版本依赖。
  - **解决方案**: 将依赖配置修改为 `^2.6.3`（允许匹配主版本下最新的兼容版本）。此外，这也提示开发者在修改完云函数代码后，**必须在微信开发者工具中右键点击云函数目录 -> 选择“上传并部署：云端安装依赖（不上传node_modules）”**，这样腾讯云服务器才会自动帮您安装最新的 sdk，彻底解决该模块找不到的报错。

- **修复饮食记录删除功能未生效的问题**:
  - **问题现象**: 用户点击“删除”食物记录时，界面提示已删除，但刷新页面或重新进入后该食物仍存在。
  - **解决方案**: 在前端 `miniprogram/packages/resources/food-library/index.js` 中解除了调用 `deleteRecord` 云函数接口的注释封印，现在前端已经成功连通了后台删除能力，并补充了防卡顿的异步延迟加载策略。

- **修复自定义食物默认保存为低嘌呤的问题**:
  - **问题现象**: 用户在添加自定义食物时，无论选择“中嘌呤”还是“高嘌呤”，最终保存到列表和数据库中的记录统统显示为“低嘌呤”。
  - **根本原因**: 在 WXML 中绑定数据参数时使用了驼峰命名 `data-purineLevel="xxx"`，而在微信小程序底层的 dataset 解析机制中，所有的驼峰字母会被强制转换为全小写（即接收到的变成了 `purinelevel`），导致 JS 代码中解构 `purineLevel` 失败，最终触发了 `|| 'low'` 的兜底逻辑。
  - **解决方案**: 将 WXML 中的所有传参属性修改为微信官方推荐的连字符格式 `data-purine-level="xxx"`。微信底层会自动将其转回小驼峰 `purineLevel` 供 JS 正确接收。现已能够精确录入用户的自定义嘌呤选项。

### 修复 (Fixed)
- **建立健康档案跳转修复**:
  - **问题**: 在 `pages/health-info/init` 页面建立档案成功后，无法跳转到病历夹首页。
  - **根本原因**: 使用了 `wx.redirectTo` 跳转到 `pages/health-info/index`，但该页面是 tabBar 页面，只能使用 `wx.switchTab`。
  - **解决方案**: 将跳转方法修改为 `wx.switchTab`。

- **病历夹功能跳转问题修复**:
  - **问题**: 从首页点击"病历夹"无法打开页面
  - **根本原因**:
    - `pages/health-info/index` 是 tabBar 页面，但首页使用了 `wx.navigateTo()` 跳转，应该使用 `wx.switchTab()`
    - `onLoad` 和 `onShow` 生命周期都执行跳转检查，可能导致重复跳转冲突
    - 使用 `setTimeout` 延迟跳转，体验不佳
  - **解决方案**:
    - 修复 `pages/home/index.js` 中的 `handleFeatureTap()` 方法，判断目标页面是否为 tabBar 页面
    - 如果是 tabBar 页面（`/pages/health-info/index`），使用 `wx.switchTab()` 跳转
    - 如果是普通页面，继续使用 `wx.navigateTo()` 跳转
    - 优化 `pages/health-info/index.js` 的生命周期逻辑：
      - `onLoad()`: 调用 `checkAndLoadProfile()`，只检查一次是否需要跳转到初始化页面
      - `onShow()`: 调用 `loadProfileData()`，只刷新数据，不进行跳转检查
    - 移除 `setTimeout` 延迟，直接使用 `wx.redirectTo()` 跳转
    - 为所有页面跳转操作添加 `fail` 回调，提供错误处理和用户提示
  - **修改文件**:
    - `miniprogram/pages/home/index.js` - 修复首页跳转逻辑
    - `miniprogram/pages/health-info/index.js` - 优化生命周期和跳转逻辑
  - **测试验证**:
    - 从未建立档案的用户首次点击"病历夹"，正常跳转到初始化页面
    - 已建立档案的用户点击"病历夹"，正常显示十二宫格菜单
    - 从其他页面返回病历夹页面时，数据正确刷新
    - 点击菜单项进入详情页正常工作
