# 医院宣教知识库 - 数据库配置教程

本教程将指导您如何在腾讯云开发 (CloudBase) 中配置医院宣教知识库的数据集合，以便在管理端（Web）和小程序端实现数据的增删改查。

## 1. 登录云开发控制台

1. 访问并登录 [腾讯云开发控制台](https://console.cloud.tencent.com/tcb)。
2. 选择您当前项目所使用的**环境**（需与系统 `.env` 和小程序配置中的环境ID一致）。

## 2. 创建数据集合

在左侧菜单栏中点击 **「数据库」**。

### 2.1 创建 `education_articles` 集合
1. 点击 **「添加集合」** 或 **「新建集合」** 按钮。
2. 集合名称填入：`education_articles`
3. 权限设置建议：
   - 如果系统尚未实现严格的用户权限，初期可选择 **「所有用户可读，仅创建者可读写」** 或 **「所有人可读写」**。
   - **推荐权限配置（针对生产环境）**：配置自定义安全规则。医生端通过 Server 接口进行写操作，患者端（小程序）直接读取云数据库：
     ```json
     {
       "read": "true",
       "write": "false" // 客户端不可写，仅能通过管理端API写入
     }
     ```
4. 点击“确定”完成创建。

## 3. 添加索引以优化查询 (强烈推荐)

为了在后台管理及小程序端快速加载及筛选文章，需要为常用的查询字段创建索引。

在 `education_articles` 集合的详情页面，点击 **「索引管理」** -> **「添加索引」**。

添加以下三个索引：

1. **按状态和创建时间查询** (小程序端获取已发布的最新文章列表)：
   - 索引名称：`status_createdAt`
   - 索引字段：
     - `status`（升序或降序皆可）
     - `createdAt`（降序，即 `-1`）
   - 唯一性：否

2. **按分类和时间查询** (小程序端按分类筛选)：
   - 索引名称：`category_createdAt`
   - 索引字段：
     - `category`（升序或降序皆可）
     - `createdAt`（降序，即 `-1`）
   - 唯一性：否

3. **按热度/浏览量查询** (小程序端热门推荐)：
   - 索引名称：`views_desc`
   - 索引字段：
     - `views`（降序，即 `-1`）
   - 唯一性：否

## 4. 插入测试数据

为了方便前端调试，您可以在云开发控制台中通过“添加文档”或“导入”功能，插入测试数据。

您可以使用已经为您生成的 `mock_education_articles.json` 或 `mock_education_articles_array.json` 进行一键导入。

**(已完成：当前代码中 Web 管理端和患者小程序端均已更新，全面对接 CloudBase 数据库 `education_articles` 集合进行真实数据的读写展示)**

### 4.1 `status` 字段规范（重要）

为避免患者端“看不到文章”，请优先统一使用以下状态值：

- 已发布：`已发布`
- 草稿：`草稿`

虽然当前患者端已兼容历史状态写法（如 `published` / `true` / `1` 或缺失字段），但生产环境建议尽快清洗历史数据并统一状态值，减少后续维护成本。

推荐在云数据库执行一次数据巡检：

1. 检查 `education_articles` 中是否存在 `status` 缺失或非标准值。
2. 将需要对患者可见的文章统一改为 `已发布`。
3. 将暂不展示的文章统一改为 `草稿`。

## 5. 云存储配置（用于文章封面图）

如果将来您希望直接在 Web 端上传封面图，请按照以下步骤配置文件存储：
1. 在控制台左侧菜单点击 **「云存储」**。
2. 确保环境的存储服务已开通。
3. 可选：可以在根目录下创建一个 `articles` 文件夹，专门用于存放医生端上传的文章封面和插图。
4. 权限设置建议：
   - 所有用户可读，仅创建者可读写（`auth != null` 可读写，`true` 可读）。

完成以上步骤后，您的数据库环境已准备就绪，系统即可接入真实数据。
