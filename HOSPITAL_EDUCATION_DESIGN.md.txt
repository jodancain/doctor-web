# 医院宣教知识库 - 功能设计方案书

## 一、 项目背景与目标

痛风患者的日常自我管理（饮食、服药、运动等）对病情控制至关重要。因此，在医生端管理系统（Web端）与患者端（小程序）之间建立互通的“健康知识库”功能，旨在实现：
1. **Web端（医生/运营人员）**：便捷地发布、编辑、管理痛风相关的健康科普文章。
2. **小程序端（患者）**：随时浏览高质量的科普内容，按分类检索，提高自我健康管理认知。

## 二、 角色与权限

1. **医生/管理员（Web端）**
   - 权限：发布新文章、编辑已有文章、下架（转草稿）文章、删除文章、查看文章阅读量统计。
2. **患者（小程序端）**
   - 权限：浏览已发布的科普文章、按分类筛选文章、进入详情页阅读。

## 三、 功能模块设计

### 1. Web端（医生管理后台 - 医院宣教模块）

#### 1.1 文章列表页 (EducationManagement)
- **卡片/列表展示**：通过可视化的网格卡片方式展示文章，包含：封面图、分类标签、状态（已发布/草稿）、标题、内容摘要、阅读时长、浏览次数。
- **状态管理**：支持“已发布”和“草稿”两种状态切换。草稿状态下文章不对患者可见。
- **筛选与搜索**：
  - 顶部支持按分类（全部、饮食、药物、基础、生活）快速切换。
  - 支持通过输入关键字搜索“文章标题”。
- **快捷操作**：悬浮在封面图上方显示快捷操作（编辑、发布/转草稿、删除）。

#### 1.2 文章发布/编辑页（弹窗/新建页）
- **核心字段**：
  - `文章标题`（必填，限制字数以保持列表美观）
  - `分类`（必填，单选：饮食 / 药物 / 基础 / 生活）
  - `预计阅读时间`（选填，单位：分钟，提升患者体验）
  - `发布状态`（立即发布 / 存为草稿）
  - `封面图片URL`（支持图床链接填入）
  - `文章正文`（富文本编辑/Markdown，目前MVP版采用多行文本输入，支持基础排版）

### 2. 小程序端（患者展示页 - 健康知识库）

#### 2.1 知识库首页
- **分类导航栏**：横向滚动的分类标签（全部、饮食、药物、基础、生活）。
- **热门推荐卡片（Banner）**：
  - 展示最新或置顶的重点宣教内容（如：痛风最新指南等），吸引患者点击。
  - 显示大图封面、标题及文章摘要。
- **最新文章列表（List）**：
  - 纵向滑动列表。
  - 每行显示文章标题、分类徽标、阅读量统计及预计阅读时间。

#### 2.2 文章详情页
- 顶部展示大封面（可选）。
- 标题、发布时间、阅读量。
- 正文内容渲染（支持富文本/HTML）。
- 底部相关推荐（下一篇）。

## 四、 数据库/数据结构设计 (参考)

**集合名称**: `education_articles`

| 字段名 | 类型 | 描述 | 必填 | 默认值 |
| :--- | :--- | :--- | :--- | :--- |
| `_id` | String | 唯一标识 | 是 | 自动生成 |
| `title` | String | 文章标题 | 是 | - |
| `category` | String | 分类 (饮食/药物/基础/生活) | 是 | '基础' |
| `content` | String | 正文内容 | 是 | - |
| `coverUrl` | String | 封面图地址 | 否 | '' |
| `readTime` | Number | 预计阅读时长(分钟) | 否 | 3 |
| `views` | Number | 浏览次数 | 否 | 0 |
| `status` | String | 状态 (已发布/草稿) | 是 | '草稿' |
| `authorId` | String | 创建者/医生ID | 是 | - |
| `createdAt` | Date | 创建时间 | 是 | Date.now() |
| `updatedAt` | Date | 最后更新时间 | 是 | Date.now() |

## 五、 后续优化规划 (Phase 2)

1. **富文本编辑器接入**：在 Web 端引入 Quill 或 WangEditor 替代 Textarea，支持图文混排。
2. **封面图直传功能**：支持本地拖拽上传图片直接存入云开发 Cloud Storage。
3. **针对性推送**：结合患者的检测数据，在小程序端为患者做个性化的“智能推荐”文章（如尿酸高频超标的患者重点推荐药物与饮食文章）。
4. **互动功能**：患者阅读后可进行“收藏”、“有用”评价，用于进一步分析文章质量。
