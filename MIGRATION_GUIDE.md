# 痛风管家小程序迁移指南 (Migration Guide)

本指南详细说明了将“痛风管家”小程序迁移到新微信账号（AppID）及新云开发环境的完整流程。

## 1. 准备工作

### 1.1 注册新账号
*   登录 [微信公众平台](https://mp.weixin.qq.com/)。
*   注册一个新的小程序账号（如果已有，直接登录）。
*   获取新的 **AppID**（在“开发” -> “开发管理” -> “开发设置”中）。

### 1.2 开通云开发
*   在微信开发者工具中，使用新账号登录。
*   点击工具栏的“云开发”按钮，开通云开发服务。
*   创建一个新的云环境（建议命名为 `prod` 或 `release`）。
*   获取新的 **环境 ID**（例如：`cloudbase-8giu16pj19de5d78`）。

## 2. 代码配置更新

### 2.1 更新项目配置
打开 `project.config.json`，修改 `appid` 字段为新账号的 AppID：

```json
{
  "appid": "wx_NEW_APPID_HERE",
  "projectname": "wechatcode-new",
  ...
}
```

### 2.2 更新云环境 ID
打开 `miniprogram/app.js`，将 `env` 字段更新为新环境 ID：

```javascript
// miniprogram/app.js
App({
  globalData: {
    // 更新为新的环境 ID
    env: 'cloudbase-8giu16pj19de5d78', 
  },
  ...
})
```

## 3. 云数据库迁移

由于云开发环境之间的数据不互通，需要手动迁移数据。

### 3.1 导出旧数据
1.  登录旧账号的微信开发者工具 -> 云开发控制台 -> 数据库。
2.  依次选择以下集合，点击“导出”（推荐 JSON 格式）：
    *   `uaRecords` (尿酸记录)
    *   `attackRecords` (发作记录)
    *   `dietRecords` (饮食记录)
    *   `waterRecords` (饮水记录)
    *   `exerciseRecords` (运动记录)
    *   `medicationReminders` (用药提醒)
    *   `food-library` (食物库 - 如果有自定义数据)
    *   `medicalProfile` (病历夹档案)

### 3.2 导入新环境
1.  切换到新账号的云开发控制台 -> 数据库。
2.  **新建集合**：按上述列表手动创建所有集合名称。
3.  **导入数据**：进入每个集合，点击“导入”，选择之前导出的 JSON 文件。
    *   *注意*：如果数据量较大，建议分批导入。
    *   *关键*：导入后，数据的 `_openid` 字段仍为旧用户的 OpenID。如果迁移意味着用户也变了（例如从测试号迁到正式号，用户重新登录），则旧数据在新环境下将无法被普通用户查询到（因为权限通常是 `仅创建者可读写`）。
    *   **策略**：如果是全量迁移且用户不变（同主体迁移），需通过后台脚本批量更新 `_openid`；如果是新项目启动，建议仅导入公共数据（如 `food-library`），用户数据清空重新录入。

### 3.3 设置数据库权限
为每个集合设置访问权限（通常在“数据权限”选项卡中）：
*   `food-library`: 所有用户可读，仅管理者可写。
*   其他用户记录表 (`uaRecords` 等): 仅创建者可读写（或自定义安全规则）。

## 4. 云函数部署

### 4.1 安装依赖
在本地终端中，进入每个云函数目录并安装依赖（如果 `node_modules` 未上传）：

```bash
cd cloudfunctions/healthDataService
npm install
```

### 4.2 上传部署
1.  在微信开发者工具中，右键点击 `cloudfunctions` 文件夹。
2.  选择当前环境为新环境。
3.  右键点击 `healthDataService`（以及其他云函数），选择“上传并部署：云端安装依赖”。

## 5. 云存储迁移 (可选)

如果旧项目中使用了云存储（如用户上传的图片、静态资源）：
1.  从旧环境云存储下载文件。
2.  上传到新环境的云存储对应目录。
3.  **注意**：代码中如果硬编码了 `fileID` (以 `cloud://` 开头)，需要替换为新环境的 `fileID`。建议使用相对路径或动态获取。

## 6. 测试与验证

1.  **编译运行**：在开发者工具中重新编译项目。
2.  **功能检查**：
    *   首页数据加载是否正常？
    *   尝试添加一条尿酸记录，检查云数据库是否新增数据。
    *   检查“我的”页面登录功能是否正常。
3.  **真机调试**：使用手机预览，确保云函数调用畅通。

## 7. 发布上线

1.  在开发者工具中点击“上传”，提交代码版本。
2.  登录微信公众平台，提交审核。
3.  审核通过后，发布上线。

---
**附录：常见问题**
*   **Q: 为什么数据读不到？**
    *   A: 检查 `app.js` 中的 `env` 是否正确，以及云数据库的权限设置是否为“仅创建者可读写”。
*   **Q: 云函数调用失败？**
    *   A: 检查云函数是否已部署到新环境，且本地调试基础库版本是否匹配。
