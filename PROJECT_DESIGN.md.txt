# 科研项目管理系统设计方案 (V1.0)

## 1. 核心目标
为医生提供一个完整的临床科研项目管理工具，支持从项目创建、受试者招募、到CRF（临床病例报告表）设计的全流程管理。

## 2. 数据库设计 (CloudBase)

### 2.1 科研项目集合 (\esearchProjects\)
存储科研项目的基本信息和配置。
- \_id\: 项目唯一标识
- \
ame\: 项目名称
- \periodValue\: 项目周期数值
- \periodUnit\: 项目周期单位 (月/年/周)
- \	argetCount\: 目标入组人数
- \enrolledCount\: 已入组人数
- \description\: 项目介绍
- \status\: 状态 (\Pending\ 设计中 / \Recruiting\ 进行中 / \Completed\ 已结束)
- \creatorId\: 创建医生ID
- \createdAt\: 创建时间

### 2.2 项目受试者集合 (\projectSubjects\)
关联患者与科研项目。
- \_id\: 记录唯一标识
- \projectId\: 关联的项目ID
- \patientId\: 关联的患者ID (OpenID)
- \joinDate\: 入组时间
- \status\: 状态 (活跃/退出/完成)

## 3. 功能模块规划

### 3.1 项目列表页 (\ProjectList\)
- **数据展示**：以列表形式展示项目名称、状态、进度条（已入组/目标人数）、周期和创建时间。
- **筛选与检索**：支持按
我的项目/全部项目过滤，按状态下拉筛选，以及按项目名称模糊搜索。
- **操作入口**：提供新建项目按钮，点击项目名称可进入项目详情/编辑页。

### 3.2 项目创建与编辑页 (\ProjectForm\)
采用分步向导式表单设计（目前实现基本信息部分）：
1. **基本信息**：项目名称、周期、目标人数、项目介绍。
2. **研究方案**：(规划中) 上传或编辑研究方案文档。
3. **CRF设计**：(规划中) 类似问卷设计，支持自定义临床数据采集表单。
4. **项目发布**：确认信息无误后，将项目状态从设计中更改为进行中。

## 4. 前端交互优化 (当前迭代)
- **状态管理**：将原先的静态 Mock 数据替换为 React State，实现项目的新增、列表展示的动态更新。
- **表单验证**：在保存项目前，对必填项（如项目名称、目标人数）进行前端校验。
- **进度可视化**：使用进度条直观展示受试者招募进度。

## 5. 后续演进路径
- **对接云数据库**：将前端的 State 管理替换为真实的 CloudBase CRUD 操作。
- **CRF 表单引擎**：复用问卷系统的底层逻辑，实现 CRF 表单的动态生成与数据收集。
- **患者端对接**：在小程序端向符合条件的患者推送招募邀请，患者同意后自动入组。
