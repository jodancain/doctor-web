# UI 调整指南

## 设计基调
- 整体采用浅色系和圆角卡片风格，聊天气泡参考 iOS 常见的左右对话布局。
- 功能入口以 2-3 列卡片展示，突出「咨询医生」和「消息」等关键操作。
- **医生工作台**：采用现代简洁设计，注重信息层级和可读性，适合快速扫描。

## 新增页面
- `pages/consult`：医生列表卡片，留有头像、昵称、标签和操作按钮，可按需要增加评分、擅长领域等字段。
- `pages/chat`：左右气泡+底部输入条，可按需扩展图片、语音、快捷短语等组件。
- `pages/doctor-messages`：列表卡片展示患者昵称、最新消息与时间戳，当前样式支持文字省略，适合扩展未读数角标。
- `pages/account`：信息表单布局，按钮分为普通、强调和危险三种状态。
- `packages/consult/doctor-client`：医生工作台首页，包含医生资料、KPI卡片、待办事项等模块。

## 主题与样式
- 主色：#5B8EF1 渐变，辅色：#4F46E5、#7AA8FF；背景统一使用 #F6F8FA。
- 圆角：卡片 24rpx（统一），气泡 24rpx，按钮 16rpx。
- 字体：标题 30-34rpx，正文 26-28rpx，说明 22-24rpx。
- 阴影：卡片使用微妙阴影 `0 2rpx 12rpx rgba(0, 0, 0, 0.04)`，提升层次感。

## 医生工作台设计规范

### 头部区域
- 医生头像：88rpx，蓝色渐变背景，白色边框
- 医生姓名：34rpx，粗体，深色
- 科室信息：24rpx，中等字重，次要颜色
- 通知按钮：72rpx，圆形，带红点提示

### KPI 卡片（2×2 网格）
- 统一尺寸：最小高度 180rpx，内边距 28rpx 24rpx
- 网格间距：20rpx
- 标签：24rpx，次要颜色
- 数值：52rpx，粗体，主色
- 高危预警卡片：左侧红色边框（4rpx），浅红背景渐变，突出但不激进

### 待办事项
- 卡片高度：最小 120rpx，内边距 20rpx 24rpx
- 左侧指示条：6rpx，红色（紧急）或蓝色（普通）
- 标题：28rpx，中等字重
- 描述：24rpx，次要颜色，行高 1.5
- 操作按钮：60rpx 高度，蓝色渐变，圆角 16rpx

### 底部导航栏
- 高度：120rpx（含安全区域）
- 图标：44rpx，选中时放大 1.05 倍
- 文字：20rpx，选中时加粗并变蓝
- 边框：顶部 1rpx，浅色

## 动效与状态
- 按钮支持 `loading` 态，列表加载失败给出 `wx.showToast` 反馈。
- 聊天输入框保留发送失败提示位，后续可引入重发或消息状态标记。
- 卡片点击：`transform: scale(0.98)`，轻微缩放反馈
- 过渡动画：`transition: all 0.2s ease`，统一动画时长

## 可扩展点
- 在 `doctor-card`、`inbox-card` 组件化后可复用样式，便于 A/B 测试。
- 预留 `messages` 集合的 `createdAt` 时间字段，可接入长轮询或 WebSocket 同步方案。
- 账号中心可增加头像上传、实名认证等入口。
- KPI 卡片可扩展更多数据可视化（图表、趋势线等）。
