# 问卷系统全链路设计方案 (V2.0)

## 1. 整体架构闭环
为了实现问卷从
医生设计到患者填写再到结果分析的完整闭环，系统架构设计如下：
- **医生端 (Web)**：负责问卷的动态设计、发布、生成分发链接/小程序码、定向下发给特定患者、导出空白文档，以及查看患者的填写结果。
- **云开发数据库 (CloudBase)**：作为数据中枢，存储问卷模板 (\questionnaires\ 集合)、患者答卷记录 (\questionnaireRecords\ 集合) 以及问卷下发任务 (\patientTasks\ 集合)。
- **患者端 (微信小程序)**：接收问卷下发通知（微信订阅消息/站内信），动态渲染问卷表单，收集患者输入并提交至数据库。

## 2. 数据库设计 (CloudBase)

### 2.1 问卷模板集合 (\questionnaires\)
存储医生设计的问卷结构。
- \_id\: 问卷唯一标识
- \	itle\: 问卷标题
- \	ype\: 问卷类型 (Scale 量表 / Survey 调查)
- \status\: 状态 (Draft 草稿 / Published 已发布)
- \questions\: 题目数组 (包含题目类型、标题、必填项、选项等)
- \creatorId\: 创建医生ID
- \createdAt\ / \updatedAt\: 时间戳

### 2.2 问卷下发任务集合 (\patientTasks\)
用于记录医生定向发给患者的问卷任务。
- \_id\: 任务唯一标识
- \_openid\: 接收任务的患者微信 OpenID
- \	askType\: 任务类型 (固定为 'questionnaire')
- \eferenceId\: 关联的问卷ID (\questionnaireId\)
- \	itle\: 任务标题（如：请填写：痛风饮食习惯调查）
- \status\: 状态 (pending 待填写 / completed 已完成)
- \createdAt\: 下发时间
- \completedAt\: 完成时间

### 2.3 答卷记录集合 (\questionnaireRecords\)
存储患者的填写结果。
- \_id\: 答卷唯一标识
- \questionnaireId\: 关联的问卷ID
- \	askId\: 关联的任务ID（如果是通过定向下发填写的）
- \_openid\: 填写患者的微信 OpenID
- \nswers\: 答题明细数组 (记录每道题的选中项或文本)
- \score\: 总分 (针对量表)
- \esult\: 结论摘要 (根据得分自动生成或医生补充)
- \submittedAt\: 提交时间戳

## 3. 医生端 (Web) 功能升级规划

### 3.1 分发与定向下发
当问卷状态为已发布时，提供以下分发方式：
1. **定向下发 (推送通知)**：医生在患者详情或问卷管理页面，选择特定患者发送问卷。系统会在 \patientTasks\ 集合中生成一条任务记录，并触发微信小程序的**订阅消息 (Subscribe Message)** 推送给该患者。
2. **小程序路径**：生成类似 \pages/questionnaire/detail?id=Q001\ 的路径，医生可直接复制发送给患者。
3. **小程序码**：调用微信接口，生成带有问卷ID参数的小程序码，支持下载后打印或发送到微信群。

### 3.2 文档生成与导出
1. **导出空白问卷 (PDF/Word)**：将 JSON 结构的问卷转换为排版良好的 PDF 或 Word 文档，供线下纸质填写。
2. **导出答卷数据 (Excel/CSV)**：在问卷记录页面，支持将收集到的患者答卷批量导出，便于科研和统计分析。

## 4. 患者端 (微信小程序) 功能规划

### 4.1 消息通知与待办中心
- **微信订阅消息**：患者在小程序授权接收服务通知后，当医生下发问卷时，患者的微信聊天列表会收到一条服务通知（如：您有一份新的健康问卷待填写）。
- **待办任务列表 (To-Do List)**：小程序首页 (Dashboard) 新增我的任务或待办事项模块，实时拉取 \patientTasks\ 集合中 \status: 'pending'\ 的问卷任务，提示患者点击填写。

### 4.2 动态问卷渲染引擎
- 新增页面：\pages/questionnaire/detail\
- **逻辑**：页面加载时获取 URL 参数中的 \id\ (问卷ID) 和 \	askId\ (任务ID)，向云数据库请求对应的问卷 JSON 结构。
- **UI 渲染**：根据 \questions\ 数组中的 \	ype\ (单选、多选、问答、评分)，动态渲染对应的原生小程序表单组件。

### 4.3 提交与反馈
- 患者提交前进行必填项校验。
- 提交后，将数据写入 \questionnaireRecords\ 集合。
- **状态同步**：如果带有 \	askId\，则同时将 \patientTasks\ 集合中对应任务的状态更新为 \completed\。
- 如果是量表类型，前端或云函数自动计算得分，并向患者展示初步的健康建议或结果反馈。

## 5. 实施路径
- **第一阶段**：完善医生端 UI，增加分享/分发和导出文档的入口。（已在UI层面更新）
- **第二阶段**：在腾讯云 CloudBase 中建立对应的集合 (\questionnaires\, \patientTasks\, \questionnaireRecords\)。
- **第三阶段**：开发医生端的定向下发功能，并对接微信小程序的订阅消息下发 API。
- **第四阶段**：开发微信小程序端的任务列表页、动态问卷渲染页面及提交逻辑。
- **第五阶段**：对接微信服务端 API，实现小程序码的真实生成；引入 \jspdf\ 或 \html2canvas\ 实现前端文档导出。
