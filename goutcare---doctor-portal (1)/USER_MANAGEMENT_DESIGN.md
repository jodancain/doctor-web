# 用户管理功能 - 设计与实现文档

## 1. 业务背景
在医疗管理系统中，除了医生本人外，往往需要多角色协同工作（如主任医师、主治医生、护士、科室管理员等）。为了保证数据安全和职责分离，系统需要一套完善的**用户管理（User Management）**功能，以便对使用该Web后台的用户进行生命周期管理。

## 2. 核心功能

### 2.1 用户列表 (User List)
*   **可视化呈现**：使用表格展示系统内的用户账号，包含名称、编码/ID、所属组织（院区/科室）、状态（启用/停用）、最后更新时间。
*   **多维度检索与筛选**：
    *   **快速搜索**：支持通过用户名称或编码进行模糊匹配。
    *   **组织筛选**：支持通过下拉框快速筛选“全部院区”或特定医院/科室（如：南方医科大学珠江医院）。

### 2.2 用户生命周期操作
*   **新增用户 (Create)**：通过弹窗表单录入新用户，包含必填项（名称、编码/ID）及选填项（所属组织、状态配置）。
*   **编辑用户 (Update)**：在列表中点击“编辑”图标，回显用户当前信息，支持修改用户的组织归属、姓名等。
*   **状态启停 (Toggle Status)**：支持一键切换用户的“启用/停用”状态。停用后的用户将无法登录系统，但其历史操作数据予以保留。
*   **删除用户 (Delete)**：支持对误建或彻底作废的用户进行删除操作（需二次确认防止误删）。

## 3. 前端交互设计 (UI/UX)
*   **操作按钮悬浮显示 (Hover Actions)**：为保持表格清爽，将“编辑”、“删除”等操作按钮设计为鼠标悬浮在对应行时才透明度渐变显示，降低视觉噪音。
*   **状态徽标 (Status Badges)**：使用色彩心理学设计状态标签。
    *   `启用`：绿色背景与边框（成功、通行）。
    *   `停用`：灰色背景与边框（失效、静止）。
*   **无数据占位符 (Empty State)**：当搜索无结果或列表为空时，展示友好的空状态提示。
*   **响应式布局**：工具栏（新增按钮、搜索框、筛选下拉框）在移动设备或小屏幕上会自动折叠为纵向排列，保证可用性。

## 4. 数据结构设计参考

在后续对接数据库（CloudBase / MongoDB 等）时，建议使用如下用户表结构（`system_users`）：

| 字段名 | 类型 | 描述 | 约束 |
| :--- | :--- | :--- | :--- |
| `_id` | String | 内部唯一主键 | 必填 |
| `name` | String | 用户显示名称 | 必填 |
| `code` | String | 登录账号 / 工号 | 必填，唯一 |
| `org` | String | 所属组织/院区标识 | 选填 |
| `status` | String | 状态 (`启用` / `停用`) | 必填，默认`启用` |
| `role` | String | 角色标识 (关联Role表) | 选填 |
| `createdAt` | Date | 账号创建时间 | 自动生成 |
| `updatedAt` | Date | 最后修改时间 | 自动更新 |
