# 云函数故障排查指南

## 概述

本文档提供云函数部署和使用过程中的常见问题及解决方案。

## 常见错误及解决方案

### 1. 错误：未找到指定的Function

**错误信息**：
```json
{
    "Response": {
        "Error": {
            "Code": "ResourceNotFound.Function",
            "Message": "未找到指定的Function，请创建后再试。"
        },
        "RequestId": "xxx"
    }
}
```

**问题原因**：
1. `config.json` 文件缺失（最常见）
2. 云函数目录结构不完整
3. 项目配置错误
4. 云环境配置问题

**解决步骤**：

#### 步骤1：检查并创建 config.json

确认 `cloudfunctions/healthDataService/config.json` 文件存在，如果不存在，创建该文件：

```json
{
  "permissions": {
    "openapi": [
      "wxacode.get"
    ]
  },
  "triggers": []
}
```

**文件位置**：
```
cloudfunctions/
└── healthDataService/
    ├── index.js
    ├── package.json
    ├── config.json  ← 必须存在
    └── src/
```

#### 步骤2：检查目录结构

确认云函数目录包含以下文件：

```
cloudfunctions/healthDataService/
├── index.js          ✅ 入口文件
├── package.json      ✅ 依赖配置
├── config.json       ✅ 权限配置（必须）
└── src/
    ├── handler.js    ✅ 请求处理器
    ├── services/     ✅ 业务服务
    └── utils/        ✅ 工具函数
```

#### 步骤3：检查项目配置

确认 `project.config.json` 配置正确：

```json
{
  "miniprogramRoot": "miniprogram/",
  "cloudfunctionRoot": "cloudfunctions/",  ← 确认此项
  ...
}
```

#### 步骤4：检查云环境配置

确认 `miniprogram/app.js` 中的环境配置：

```javascript
App({
  globalData: {
    env: 'your-env-id',  // 或留空使用默认环境
  },
  onLaunch() {
    wx.cloud.init({
      traceUser: true,
      env: this.globalData.env  // 如果设置了env
    });
  }
});
```

#### 步骤5：重新部署

1. 在云开发控制台删除旧函数（如果存在）
2. 在微信开发者工具中右键 `cloudfunctions/healthDataService`
3. 选择「上传并部署：云端安装依赖」
4. 等待部署完成

### 2. 错误：环境ID无效

**错误信息**：
```
Error: -501000 Param Invalid: env check invalid
```

**解决方案**：
- 检查 `miniprogram/app.js` 中的 `env` 配置
- 确认环境ID与云开发控制台中的环境ID一致
- 如果环境ID不正确，可以留空使用默认环境

### 3. 错误：权限不足

**错误信息**：
```
Error: Permission denied
```

**解决方案**：
1. 检查 `config.json` 中的权限配置
2. 检查数据库集合权限设置
3. 确认云函数已正确部署

### 4. 错误：依赖安装失败

**错误信息**：
```
npm install failed
```

**解决方案**：
1. 检查 `package.json` 中的依赖配置
2. 确认网络连接正常
3. 尝试使用「上传并部署：全量文件」方式

### 5. 常见代码错误排查

### 错误：查询结果为空（但数据库有数据）

**现象**：
- 数据库中有数据，但小程序端列表显示为空。
- 调试日志显示 `records` 为空数组。

**原因**：
- `miniprogram/utils/request.js` 中的 `call` 函数已经处理了返回结果，直接返回了 `result.data`。
- 如果在业务代码中再次尝试解构 `result.data`，会导致 `undefined`。

**错误代码示例**：
```javascript
const result = await call('getRecords', ...);
const records = result.data; // ❌ 错误：result 已经是数组了，result.data 是 undefined
```

**正确代码示例**：
```javascript
const result = await call('getRecords', ...);
// 兼容处理
const records = Array.isArray(result) ? result : ((result && result.data) ? result.data : []);
```

### 错误：module is not defined

**现象**：
- 报错 `Error: module 'packages/services/chat.js' is not defined`。

**原因**：
- `require` 引用路径错误。例如使用 `../../services/chat` 时层级计算错误，指向了不存在的目录。

**解决方案**：
- 检查文件相对路径。例如在 `packages/consult/doctor-client/index.js` 中引用 `packages/consult/services/chat.js`，应该使用 `../services/chat` 而不是 `../../services/chat`。

### 6. 渲染层网络错误 (Failed to load image)

**现象**：
- 控制台报错 `[渲染层网络层错误] Failed to load image ... the server responded with a status of 400`。
- 头像或图片无法显示。

**原因**：
- 图片 URL 失效、过期或访问受限（如防盗链）。
- 默认头像使用了不稳定的网络图片链接。

**解决方案**：
- **替换为本地资源**：将默认头像替换为本地图片（如 `/images/default-avatar.png`）。
- **使用 CSS 绘制**：使用 `view` 标签和 CSS 样式绘制纯色头像，显示名字首字，避免网络依赖。
  ```html
  <view class="avatar" wx:if="{{!url}}">{{name[0]}}</view>
  <image class="avatar" src="{{url}}" wx:else></image>
  ```

## 部署检查清单

在部署云函数前，请确认：

- [ ] `cloudfunctions/healthDataService/config.json` 文件存在
- [ ] `cloudfunctions/healthDataService/index.js` 文件存在
- [ ] `cloudfunctions/healthDataService/package.json` 文件存在
- [ ] `cloudfunctions/healthDataService/src/` 目录存在
- [ ] `project.config.json` 中 `cloudfunctionRoot` 配置正确
- [ ] 已登录微信开发者工具
- [ ] 已开通云开发
- [ ] 已选择正确的云环境

## 部署后验证

部署成功后，请进行以下验证：

1. **检查云函数状态**：
   - 打开云开发控制台 →「云函数」
   - 确认 `healthDataService` 状态为"正常"

2. **测试云函数调用**：
   - 在小程序端测试数据读写功能
   - 查看是否正常返回数据

3. **查看云函数日志**：
   - 在云开发控制台 →「云函数」→「日志」
   - 确认无错误信息

## 预防措施

为避免部署失败，建议：

1. **保持目录结构完整**：
   - 不要删除 `config.json` 文件
   - 保持标准的目录结构

2. **版本控制**：
   - 将 `config.json` 纳入版本控制
   - 确保团队成员都有完整的文件

3. **文档同步**：
   - 更新代码时同步更新文档
   - 记录配置变更

## 获取帮助

如果以上方法都无法解决问题：

1. 查看云开发控制台的详细错误日志
2. 检查微信开发者工具的控制台输出
3. 参考微信云开发官方文档
4. 联系技术支持
