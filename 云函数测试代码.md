# 云函数测试指南

为了确保云函数 `healthDataService` 功能的稳定性和正确性，建议在部署前进行充分的本地和云端测试。

## 1. 登录功能测试

用于测试用户登录逻辑，包括账号密码验证和自动注册流程。

**测试参数 (Template):**

```json
{
  "action": "login",
  "username": "testuser",
  "password": "password123"
}
```

**预期结果:**
- 成功：返回用户信息对象，包含 `_id`, `username`, `role` 等字段。
- 失败：返回错误码和错误信息（如 "User not found" 或 "Account already bound"）。

## 2. 获取医生列表

测试医生信息检索功能，用于前端展示。

**测试参数:**

```json
{
  "action": "listDoctors"
}
```

**预期结果:**
- 返回一个包含所有角色为 `doctor` 的用户数组。

## 3. 发送消息测试

模拟发送聊天消息，验证数据库写入和字段完整性。

**测试参数:**

```json
{
  "action": "sendMessage",
  "targetOpenid": "oX-xxxxxxxxxxxx",  // 替换为真实的接收者 OpenID
  "content": "Hello, this is a test message.",
  "msgType": "text",
  "clientMsgId": "test-msg-id-001"
}
```

**预期结果:**
- 返回新创建的消息对象，包含 `_id`, `createdAt`, `status: 'success'`。

## 4. 获取会话列表 (Inbox)

测试聚合查询，验证是否能正确按时间倒序返回最近的会话。

**测试参数:**

```json
{
  "action": "listInbox"
}
```

**预期结果:**
- 返回一个数组，每个元素代表一个会话，包含 `partnerOpenid`, `partnerName`, `latestMessage` 等摘要信息。

## 5. 图片消息处理测试

验证图片消息的发送逻辑（注意：云函数本身不处理文件上传，只处理消息记录）。

**测试参数:**

```json
{
  "action": "sendMessage",
  "targetOpenid": "oX-xxxxxxxxxxxx",
  "content": "cloud://path/to/image.jpg",
  "msgType": "image"
}
```

**预期结果:**
- 数据库中 `messages` 集合新增一条记录，`msgType` 为 `image`，`content` 为 fileID。

## 6. 常见问题排查

- **云端无法获取用户登录态信息**：
  - 确保在微信开发者工具中点击了“云端测试”或“本地调试”，并且工具已登录。
  - 如果是本地 Node.js 脚本调用，需要正确配置 `wx-server-sdk` 的密钥环境。
- **超时**：
  - 检查网络连接。
  - 检查云函数内存配置和超时时间设置（默认 3秒可能不足以进行复杂的冷启动操作）。
