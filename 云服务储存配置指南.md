# 云服务存储配置与性能优化指南

## 1. 背景与目标
为了提升小程序的用户体验，降低流量消耗，并确保云存储资源的高效利用，我们需要对云存储配置进行系统性的优化。特别是在图片密集型的场景（如聊天、病历记录）中，合理的配置能显著提升加载速度。

## 2. 图片存储优化策略

### 2.1 开启 CDN 加速 (默认开启)
腾讯云 COS（云存储底层）默认提供 CDN 加速。
- **配置检查**：确保云存储权限设置为“所有用户可读”（对于公开资源）或配置好签权访问（对于私有资源），以便利用 CDN 节点分发。

### 2.2 图片处理 (Image Processing)
利用云端实时处理能力，减少客户端下载的数据量。

#### 推荐配置规则
建议在微信开发者工具 -> 云开发 -> 存储 -> 图片处理 中配置以下样式，或者在代码中使用动态参数（当前项目策略）。

| 场景 | 样式名/参数 | 描述 | 参数详情 |
| :--- | :--- | :--- | :--- |
| **列表缩略图** | `thumbnail` | 用于聊天列表、历史记录预览 | `imageMogr2/thumbnail/400x/quality/80/interlace/1` |
| **全屏预览** | `preview` | 用于点击查看大图 | `imageMogr2/quality/85/interlace/1` (可选，适当压缩大图) |
| **头像** | `avatar` | 用于用户头像显示 | `imageMogr2/thumbnail/200x200/gravity/center/crop/200x200` |

### 2.3 缓存策略 (Cache-Control)配置指南

合理的缓存配置能大幅减少重复的网络请求，节省流量并提升二次加载速度。

#### A. 配置入口
微信开发者工具 -> 云开发 -> 存储 -> 存储配置 -> 缓存配置 -> 添加配置

#### B. 推荐配置方案

| 文件类型 | 缓存时间 | 建议值 | 理由 |
| :--- | :--- | :--- | :--- |
| **静态资源 (Avatar, Banner)** | 长期 | `30天` (43200分钟) | 用户头像、活动Banner等更新频率低，长期缓存。 |
| **聊天图片 (Chat Images)** | 永久/长期 | `1年` (525600分钟) | 聊天历史图片一旦发送不可修改，建议永久缓存。 |
| **病历记录 (Medical Records)** | 长期 | `30天` | 历史病历图片通常不会变更。 |
| **普通文件 (Documents)** | 中期 | `7天` | 可能存在版本更新的文件。 |

#### C. 具体操作步骤
1.  **类型**：选择“文件后缀”或“文件夹”。
    *   *建议按文件夹管理*，例如针对 `/chat_images` 目录设置长期缓存。
    *   *若按后缀*，输入 `jpg;png;jpeg;webp;gif`。
2.  **缓存时间**：根据上述表格填入对应数值。
    *   例如：`30` 天。
3.  **生效范围**：CDN 节点及客户端浏览器缓存。

#### D. 注意事项
- **文件名哈希**：设置长期缓存的前提是文件名必须唯一（如包含时间戳或UUID）。如果同名文件内容会更新，请**不要**设置过长的缓存时间，否则用户可能看到旧图。
- **强制刷新**：如果必须更新已缓存的文件，通常只能通过更改文件名（如 `avatar_v2.jpg`）来强制客户端重新加载。

## 3. 文件上传优化

### 3.1 客户端压缩
前端在调用 `wx.chooseImage` 或 `wx.chooseMedia` 时，务必开启压缩选项。

```javascript
wx.chooseMedia({
  mediaType: ['image'],
  sizeType: ['compressed'], // 开启压缩
  // ...
})
```

### 3.2 路径规范化
为了便于管理和后续可能的生命周期管理（如自动清理过期文件），建议采用结构化的存储路径：

- `chat_images/{yyyy}/{mm}/{timestamp}_{random}.{ext}`：聊天图片
- `avatars/{userid}_{timestamp}.{ext}`：用户头像
- `records/{record_type}/{userid}/{timestamp}.{ext}`：病历记录

## 4. 性能优化代码实践

### 4.1 预加载 (Preconnect / Preload)
虽然小程序限制了 `<link>` 标签的使用，但可以通过在页面加载早期（`onLoad`）预先获取关键图片的临时链接 (`wx.cloud.getTempFileURL`) 来减少渲染时的等待。

### 4.2 懒加载 (Lazy Load)
对于长列表（如聊天记录），必须为 `<image>` 组件开启 `lazy-load` 属性。

```html
<image src="{{url}}" lazy-load="{{true}}" mode="aspectFill" />
```

## 5. 未来扩展预留

### 5.1 视频存储
- **预留路径**：`chat_videos/`
- **处理建议**：视频上传前应在客户端或云函数端获取第一帧作为封面图，列表仅加载封面，点击后播放视频。

### 5.2 文档存储
- **预留路径**：`documents/`
- **优化**：对于 PDF/Word 等文档，不直接在移动端下载查看，而是建议利用云函数的文档转图片服务（如有）生成预览图，或使用小程序的文件预览接口。

### 5.3 生命周期管理 (Lifecycle)
- 建议定期（如每月）运行云函数脚本，清理 `chat_images` 中超过一定期限（如 1年）的临时聊天图片，或将其归档至冷存储以降低成本。
