# 云服务图片预处理配置指南

## 1. 背景与目的
为了提升聊天功能中图片的加载速度并节省用户流量，我们需要利用微信云开发（基于腾讯云 COS）提供的**图片处理（Image Processing）**能力。
通过在云端实时处理图片（缩放、压缩、格式转换），可以在不同场景下分发最合适的图片版本，而不是始终加载原图。

## 2. 图片处理方案

我们采用 **动态参数** 与 **样式封装** 相结合的策略。

### 2.1 动态处理参数 (当前代码采用)
目前代码中通过在图片 URL 后拼接参数来实现实时处理。推荐配置如下：

| 参数 | 值 | 说明 |
| :--- | :--- | :--- |
| `imageMogr2` | 开启 | 启用图片处理工具 |
| `thumbnail` | `400x` | **缩放**：限制宽度为 400px，高度自适应，兼顾清晰度与加载速度（适用于手机屏宽）。 |
| `quality` | `80` | **压缩**：将图片质量降至 80%，在肉眼难以分辨差异的情况下大幅减小体积。 |
| `interlace` | `1` | **渐进显示**：开启渐进式加载，图片会从模糊到清晰逐渐呈现，提升弱网体验。 |

**生成的 URL 示例**：
```
https://.../image.jpg?imageMogr2/thumbnail/400x/quality/80/interlace/1
```

### 2.2 控制台“图片样式”配置 (可选，推荐)
为了简化 URL 并方便统一管理，您可以在微信开发者工具的「云开发控制台」中配置**图片样式**。

#### 配置步骤：
1.  打开微信开发者工具 -> 点击工具栏「云开发」。
2.  进入「存储管理」 -> 「图片处理」。
3.  点击「添加样式」。
4.  **样式名称**：输入 `chat-list`。
5.  **缩略方式**：选择「限定宽度，高度自适应」。
6.  **宽度**：输入 `400` 像素。
7.  **输出格式**：推荐选择 `webp` (若需兼容旧iOS可选原图，但目前主流版本均支持)。
8.  **渐进显示**：开启。
9.  点击「确定」保存。

#### 代码适配：
配置完成后，代码中的 URL 拼接方式可简化为（需配合分隔符，默认通常为 `!` 或 `?`，需在控制台确认）：
```javascript
// 假设分隔符配置为 '!'
const thumbnailUrl = `${displayUrl}!chat-list`;
```

*注：由于样式配置需要手动操作控制台，为了保证代码开箱即用，当前项目代码默认使用 2.1 中的动态参数方式。*

## 3. 上传优化策略
在前端上传阶段，我们也进行了优化配置：

-   **API**: `wx.chooseImage` / `wx.chooseMedia`
-   **参数**: `sizeType: ['compressed']`
-   **效果**: 微信客户端会在上传前自动对过大的图片进行有损压缩，避免上传数 MB 的原图，显著减少上传耗时和服务器存储压力。

## 4. 预期收益
-   **列表加载速度**：提升 50% 以上（视原图大小而定）。
-   **流量消耗**：减少 60%-80%。
-   **用户体验**：配合渐进式加载和 Lazy Load，消除图片加载时的白屏和卡顿感。
