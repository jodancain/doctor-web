# 云服务配置指引

本文档描述了小程序新增的登录、医生工作台以及云端扩展能力所需的云开发配置步骤。确保已按照 `DATABASE_SETUP.md` 初始化过云开发环境后，再执行以下操作。

## 1. 数据库集合

**重要**: 详细的集合配置请参考 `DATABASE_COLLECTIONS.md` 文档。

请在云开发控制台中创建以下集合，并设置相应的权限：

| 集合名称 | 用途说明 | 推荐权限 | 关键字段 |
| --- | --- | --- | --- |
| `users` | 存储患者与医生账号 | 所有用户可读，仅创建者可写 | `_openid`, `openid`, `role`('user'/'doctor'), `nickName`, `username`, `boundPatients`, `boundDoctors` |
| `messages` | 存储聊天消息 | 仅创建者及管理端可读写 | `fromOpenid`, `toOpenid`, `content`, `msgType`, `createdAt` |
| `uaRecords` | 尿酸记录 | 仅创建者可读写 | `value`, `timestamp`, `_openid` |
| `attackRecords` | 痛风发作记录 | 仅创建者可读写 | `joint`, `severity`, `trigger`, `timestamp`, `_openid` |
| `waterRecords` | 饮水记录 | 仅创建者可读写 | `volume`, `timestamp`, `_openid` |
| `exerciseRecords` | 运动记录 | 仅创建者可读写 | `type`, `duration`, `timestamp`, `_openid` |
| `medicationReminders` | 用药提醒 | 仅创建者可读写 | `name`, `dosage`, `time`, `timestamp`, `_openid` |
| `dietRecords` | 饮食记录 | 仅创建者可读写 | `name`, `color`, `timestamp`, `_openid` |

**集合创建检查清单**:
- [ ] `users` - 用户集合
- [ ] `messages` - 消息集合  
- [ ] `uaRecords` - 尿酸记录集合
- [ ] `attackRecords` - 发作记录集合
- [ ] `waterRecords` - 饮水记录集合
- [ ] `exerciseRecords` - 运动记录集合
- [ ] `medicationReminders` - 用药提醒集合
- [ ] `dietRecords` - 饮食记录集合

> **提示**：
> - 对于 `messages` 集合，为了支持双向通信（A发给B，B也能看到），需要在云函数中处理读写，或者设置自定义安全规则允许 `fromOpenid == auth.openid || toOpenid == auth.openid` 的记录被读取。
> - `users` 集合需要设置为"所有用户可读"，以便医生列表页能拉取到医生信息，或者患者能被医生搜索到。
> - 所有记录类集合使用 `_openid` 字段进行数据隔离，确保用户只能访问自己的数据。
> - 用户绑定功能：注册和登录时会自动绑定用户的 `openid`，同时支持 `_openid`（微信云数据库自动管理）和自定义 `openid` 字段（用于兼容）。

## 2. 云函数部署

本项目使用单一云函数入口 `healthDataService`，位于 `cloudfunctions/healthDataService` 目录。

### 2.1 云函数目录结构

```
cloudfunctions/healthDataService/
├── index.js          # 入口文件
├── package.json      # 依赖配置
├── config.json       # 权限配置（必须存在）
└── src/
    ├── handler.js    # 请求处理器
    ├── services/     # 业务服务
    └── utils/        # 工具函数
```

### 2.2 部署步骤

1. **检查配置文件**：
   - 确认 `cloudfunctions/healthDataService/config.json` 文件存在
   - 如果不存在，创建该文件：
     ```json
     {
       "permissions": {
         "openapi": ["wxacode.get"]
       },
       "triggers": []
     }
     ```

2. **安装依赖**（可选）：
    在 `cloudfunctions/healthDataService` 目录下执行：
    ```bash
    npm install
    ```
   > 注意：如果选择「上传并部署：云端安装依赖」，则无需本地安装。

3. **上传部署**：
   在微信开发者工具中，右键点击 `cloudfunctions/healthDataService` 目录，选择 **"上传并部署：云端安装依赖"**。

3.  **功能说明**：
    该云函数集成了以下服务：
    - **用户服务** (`auth-service`): 处理用户注册、登录，自动绑定 `openid`
    - **用户管理** (`user-model`): 支持用户查询、医生患者绑定关系管理
    - **记录服务** (`record-service`): 处理各类健康数据的增删查改
      - 尿酸记录 (`uaRecords`)
      - 发作记录 (`attackRecords`)
      - 饮水记录 (`waterRecords`)
      - 运动记录 (`exerciseRecords`)
      - 用药提醒 (`medicationReminders`)
      - 饮食记录 (`dietRecords`)
    - **统计分析** (`statistics-service`): 提供首页汇总和数据看板分析
      - `getHomeSummary`: 获取最近尿酸值、近7日统计
      - `getDashboardData`: 获取详细数据看板（包含时间线、饮食分布等）
    - **医生服务** (`doctor-service`): 处理医生患者绑定、患者数据查看
    - **聊天服务** (`chat-service`): 处理消息发送、会话列表（如已实现）

## 3. 云函数环境变量 (可选)

`healthDataService` 默认使用 `cloud.DYNAMIC_CURRENT_ENV` 自动适配环境。通常无需手动配置环境变量。

如果需要区分测试/生产环境的特殊逻辑，可在云函数控制台为函数添加：
- `WX_ENV`: `dev` / `prod`

## 4. 医生注册说明

- 医生账号通过小程序的注册页面进行注册。
- **医生码**：注册时必须输入正确的医生码 `18736` 才能获得医生身份。
- 注册成功后，`users` 集合中的该用户记录将包含 `role: 'doctor'` 字段。

## 5. 数据来源说明

**重要**: 本系统的所有健康数据均从云数据库读取，已完全移除所有假数据和模拟数据。

- 所有健康记录（尿酸、发作、饮水、运动、用药、饮食）均通过云函数从数据库实时读取
- 统计数据（首页汇总、数据看板）均基于数据库中的真实数据计算
- 详细说明请参考 `数据来源说明.md` 文档

## 6. 常见问题排查

### 6.1 云函数上传失败：未找到指定的Function

**错误信息**：
```
Error: TencentCloud API error: {
    "Response": {
        "Error": {
            "Code": "ResourceNotFound.Function",
            "Message": "未找到指定的Function，请创建后再试。"
        }
    }
}
```

**原因分析**：
- `config.json` 文件缺失（最常见）
- 云函数目录结构不完整
- 项目配置错误

**解决步骤**：

1. **检查并创建 `config.json`**：
   - 确认 `cloudfunctions/healthDataService/config.json` 存在
   - 如果不存在，创建该文件：
     ```json
     {
       "permissions": {
         "openapi": ["wxacode.get"]
       },
       "triggers": []
     }
     ```

2. **检查目录结构**：
   - 确认包含 `index.js`、`package.json`、`config.json`
   - 确认 `src/` 目录存在且包含必要文件

3. **检查项目配置**：
   - `project.config.json` 中 `cloudfunctionRoot` 应为 `"cloudfunctions/"`
   - 确认已登录并开通云开发

4. **重新部署**：
   - 删除云开发控制台中的旧函数（如存在）
   - 右键 `cloudfunctions/healthDataService` →「上传并部署：云端安装依赖」

### 6.2 其他常见问题

- **Error: -501000 Param Invalid: env check invalid**:
  请检查 `miniprogram/app.js` 中的 `env` 配置。如果您的环境 ID 不是默认值，请将其清空或填入正确的环境 ID。

- **数据库权限错误**:
  如果前端直接读取数据库（虽然本项目主要通过云函数读取，但部分场景可能涉及），请检查控制台的"数据权限"设置。

- **消息无法发送/接收**:
  请确保 `messages` 集合已创建，且云函数已成功部署更新。

- **页面显示无数据**:
  这是正常现象，表示数据库中暂无数据。请通过相应页面添加记录，数据会自动保存到数据库。

### 6.3 部署验证

部署成功后，请验证：

1. 在云开发控制台 →「云函数」中确认 `healthDataService` 状态为"正常"
2. 在小程序端测试数据读写功能
3. 查看云函数日志确认无错误
