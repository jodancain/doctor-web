# 组件按需/用时注入规范化分析

## 背景
- 代码质量扫描显示「启用组件按需注入」检查未通过，提示需要开启按需注入能力以降低代码注入体积与耗时。
- 微信基础库自 2.11.1 起支持按需注入，自 2.11.2 起在按需注入基础上支持用时注入，通过配置可以避免未访问页面与组件在首屏被加载和执行。
- 代码质量扫描显示「主包内不应存在主包未使用的JS文件」：`services/chat.js` 被识别为未使用。

## 现状与原因分析
1.  **按需注入未生效**：
    - 当前小程序 `app.json` 仅声明了页面、窗口和 tabBar 等配置，没有开启 `lazyCodeLoading`，因此基础库在启动时会把主包和声明的分包、组件一次性注入执行，导致未访问页面与全局声明的组件都在启动阶段注入，占用额外启动时间和内存。
    - 质量扫描检测到未开启按需注入配置，导致「启用组件按需注入」项不通过。
2.  **主包包含未使用文件**：
    - `services/chat.js` 被放置在主包目录 `miniprogram/services/` 下。
    - 经分析引用关系，该文件仅被分包 `packages/consult` 中的页面（`chat/index.js`, `doctor-messages/index.js`）引用，主包无任何文件依赖它。
    - 这导致主包体积无谓增加，触发代码质量警告。

## 改进与更改建议
1.  **开启按需注入**：在 `app.json` 中添加 `"lazyCodeLoading": "requiredComponents"`，让仅当前页面依赖的自定义组件和页面代码被注入，减少首屏注入体积。（已完成调整）
2.  **主包瘦身与文件迁移**：
    - 将仅被分包引用的 `services/chat.js` 迁移至分包目录 `packages/consult/services/chat.js`。
    - 更新相关引用路径，确保主包不包含未被使用的业务逻辑代码。
3.  **精简组件声明**：
    - 清理各页面 JSON 里未实际使用的自定义组件声明，避免被视为页面依赖而提前注入。
    - 谨慎使用 `app.json` 的全局 `usingComponents`，将低使用频率组件下沉到需要的页面，提升按需注入效果。
4.  **采用用时注入（可选）**：对体积较大或低频的组件，配置占位组件以启用用时注入，在首次渲染时再加载真实组件，进一步减轻启动注入开销。
5.  **版本与验证**：
    - 调试时在开发者工具选择基础库 2.20.1 及以上版本，确保按需/用时注入生效。
    - 完成配置后逐页验证组件展示与交互，确保延迟注入不会影响首屏体验或事件绑定。

## 本次优化落实
- **主包瘦身**：将非 tabBar 且低频访问的功能（咨询/聊天、医生消息与工作台、各类健康记录、数据看板、食物嘌呤库）下沉到三个分包中，仅保留首页、登录鉴权、健康知识和账号设置在主包，彻底避免首屏下载/注入不必要的页面代码。
- **文件迁移**：将 `miniprogram/services/chat.js` 移动到 `miniprogram/packages/consult/services/chat.js`，并更新了 `packages/consult` 下相关页面的引用路径。
- **路径与样式同步**：同步更新首页入口跳转路径与分包页面对 `app.wxss` 的引用层级，确保按需注入和分包加载下的导航、样式继承正常。

## 预期收益
- 启动阶段注入和执行的 JS 代码量减少，降低首屏耗时与内存占用。
- 通过代码质量「组件按需注入」项的检测，同时为后续用时注入优化留出空间。
- 解决「主包内不应存在主包未使用的JS文件」的代码质量警告。
