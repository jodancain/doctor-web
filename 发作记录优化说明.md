# 痛风发作记录功能优化

## 优化内容

### 1. 修复图表绘制报错

**问题**：
- 报错信息：`requestAnimationFrame is not defined`
- 原因：在小程序 Canvas 2D 模式下，`requestAnimationFrame` 不是全局函数，必须通过 canvas 实例调用（如 `canvas.requestAnimationFrame`）。

**修复方案**：
- 修改 `miniprogram/packages/records/attack-record/index.js`，将全局调用改为通过 `canvas` 实例调用。

### 2. 诱因分析图表优化

**目标**：使诱因分析饼状图更接近设计稿（环形图，带间隔，圆形图例），并修复不显示的问题。

**修改详情**：

1.  **修复绘制逻辑**：
    - 添加了详细的调试日志，用于排查 Canvas 尺寸和数据问题。
    - 优化了 `gap`（间隔）的处理逻辑，防止因角度计算错误导致绘制失败。
    - 增加了对 `radius <= 0` 的防御性检查。

2.  **调整图表样式**：
    - 将实心饼图改为环形图（通过调整 `arc` 绘制逻辑）。
    - 增加扇区之间的间隔（`gap = 0.08` 弧度）。
    - 使用圆角端点（`lineCap = 'round'`）。
    - 修改绘制方式，通过 `lineWidth` 和 `stroke` 来绘制具有宽度的圆弧。

3.  **优化配色方案**：
    - 使用设计稿中的配色：
        - 饮酒：`#EF4444` (红)
        - 海鲜：`#F59E0B` (橙)
        - 受凉：`#10B981` (绿)
        - 高嘌呤饮食：`#3B82F6` (蓝)
        - 熬夜劳累：`#8B5CF6` (紫)

4.  **调整图例样式**：
    - 将图例颜色块改为圆形（`border-radius: 50%`）。
    - 调整图例间距和文字大小，使其布局更美观。

**涉及文件**：
- `miniprogram/packages/records/attack-record/index.js`
- `miniprogram/packages/records/attack-record/index.wxss`

## 故障排查：饼图不显示

如果图表仍然不显示，请在微信开发者工具的调试器（Console）中查看以下日志：

1.  **`[PieChart] Total triggers:`**
    - 如果为 0，说明没有诱因数据，图表不显示是正常的（会显示“暂无数据”）。
    - 如果大于 0，说明数据正常。

2.  **`[PieChart] Canvas size:`**
    - 检查 `width` 和 `height` 是否为正数。如果为 0，说明 Canvas 节点没撑开，检查 CSS。
    - 检查 `DPR` 是否正常（通常为 2 或 3）。

3.  **`[PieChart] Radius too small:`**
    - 如果出现此错误，说明 `width` 或 `height` 太小，导致计算出的半径无效。

### 可能的解决方案

- **清除缓存**：微信开发者工具 -> 清除缓存 -> 全部清除。
- **真机调试**：有时候模拟器的 Canvas 表现与真机不一致，尝试真机调试。
- **检查 CSS**：确保 `.pie-chart` 类正确设置了宽度和高度（`miniprogram/packages/records/attack-record/index.wxss`）。
