## 2026-03-30（第三轮 — 部署阻断 Bug 修复）

### 🔴 严重 Bug 修复

1. **【部署阻断】Dockerfile 缺少 bcrypt 编译工具** (`Dockerfile`)
   - `bcrypt` 是原生 C++ 模块，Alpine 镜像没有编译工具，Docker 构建时 `npm ci` 必定失败
   - 添加 `apk add python3 make g++`，编译完成后移除以减小镜像体积

2. **【逻辑错误】`doc().get()` 空结果检查永远为 false**（4处）
   - `db.doc(id).get()` 返回 `{ data: [] }`，空数组是 truthy，`!result.data` 永远为 false
   - 404 永远不会触发，前端收到空数组而非错误提示
   - 修复文件：`questionnaires.ts`、`projects.ts`、`content-service.cjs`（getQuestionnaire + getArticle）
   - 修复方式：`!result.data` → `!result.data || result.data.length === 0`，返回 `result.data[0]`

3. **【路由冲突】问卷回收列表路由被参数路由拦截** (`questionnaires.ts`)
   - `GET /records/list` 定义在 `GET /:id` 之后，Express 先匹配 `/:id`（id = "records"）
   - 导致问卷回收列表接口永远不可达
   - 修复方式：将 `/records/list` 路由移到 `/:id` 之前

  - `GET/POST /ai/test-suites` — 管理测试集
- **新增 `server/middleware/api-key-auth.ts`**：Bearer token API Key 认证，支持细粒度权限（patients:read / ai:test / ai:evaluate / ai:knowledge）、审计日志、`OPEN_API_ENABLED` 功能开关。
