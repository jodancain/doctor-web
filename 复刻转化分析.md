# 痛风管家-Pro 医生端复刻转化分析

## 1. 项目概述

本项目旨在完全复刻“痛风管家-Pro”的医生端 UI 排版，打造一个专业、高效、美观的慢病管理工作台。核心目标是将现有的医生端功能（数据概览、患者管理）与即时通讯（对话功能）深度集成，提供流畅的用户体验。

## 2. UI/UX 设计分析

基于《医生端开发规格说明书》及专业医疗 App 的设计规范，我们将界面风格定义为：

*   **设计语言**：现代、极简、医疗专业感。
*   **主色调**：`#059669` (Emerald Green)，传递健康、安全、信任的感觉。
*   **辅助色**：
    *   高危/警示：`#EF4444` (Red)
    *   中风险/提醒：`#F59E0B` (Amber)
    *   链接/交互：`#3B82F6` (Blue)
*   **排版原则**：
    *   **卡片式布局**：利用阴影和圆角区分内容块，增加层次感。
    *   **信息层级**：通过字号（56rpx/36rpx/28rpx/24rpx）、字重（Bold/Medium/Regular）和颜色深浅（#111/ #6B7/ #9CA）清晰区分关键数据与辅助信息。
    *   **呼吸感**：增加内边距（Padding）和元素间距（Gap），避免界面过于拥挤。

## 3. 核心页面复刻方案

### 3.1 工作台 (Workbench)
*   **顶部栏**：沉浸式设计，展示医生头像、姓名、科室及消息通知入口（带红点）。
*   **核心数据看板**：采用 2x2 网格布局，展示患者总数、达标率（带趋势）、高风险预警（高亮强调）、近期活跃数。
*   **待办事项**：垂直列表，左侧色条区分优先级（紧急/普通），右侧“处理”按钮引导操作。

### 3.2 患者管理 (Patients)
*   **顶部悬浮**：搜索栏（包含筛选图标）+ 快捷筛选标签（全部、高危、未达标、近期活跃），支持横向滚动。
*   **患者列表**：
    *   左侧头像 + 中间核心信息（姓名/性别/年龄） + 底部关键指标（UA值）。
    *   右侧状态标签（高危/偏高/正常）垂直排列，异常状态通过颜色强提示。
    *   点击跳转至患者详情页。

### 3.3 消息 (Messages) - **集成重点**
*   **UI 复刻**：复用微信聊天列表风格。
    *   左侧圆形头像（显示未读红点）。
    *   中间展示昵称 + 最后一条消息预览（截断）。
    *   右侧展示最后消息时间。
*   **功能集成**：
    *   直接嵌入在底部导航的“消息”Tab 中，而非跳转新页面。
    *   复用现有的 `chat-service`，实现轮询/WebSocket 更新。
    *   支持点击进入 `chat/index` 详情页。

### 3.4 个人中心 (Profile)
*   **头部卡片**：大头像 + 详细身份信息。
*   **功能菜单**：列表式排布（排班、评价、设置），右侧箭头引导。
*   **账户管理**：独立的退出/切换账号区域，防止误触。

## 4. 功能集成策略：对话功能

将原有的独立页面 `doctor-messages/index` 逻辑无缝迁移至主 TabBar 的“消息”视图中。

*   **数据源**：共用 `listInbox` 云函数接口。
*   **状态管理**：在 `doctor-client/index.js` 中维护 `conversations` 数据，利用 `onShow` 和 `setInterval` 实现消息的实时（或准实时）更新。
*   **交互优化**：
    *   支持下拉刷新同步最新消息。
    *   长按会话项弹出“删除”菜单。
    *   点击会话项平滑跳转至聊天详情页。

## 5. 转化实施步骤

1.  **样式重构 (Refactor Styles)**：
    *   全面应用新的配色系统和间距规范（`index.wxss`）。
    *   优化 `wxml` 结构，确保层级清晰，便于样式控制。
2.  **组件化 (Componentization)**：
    *   将消息列表项提取为模板或复用结构，确保代码整洁。
3.  **逻辑迁移 (Logic Migration)**：
    *   将 `doctor-messages` 中的轮询、删除逻辑完整迁移至 `doctor-client`。
4.  **细节打磨 (Polishing)**：
    *   头像加载失败的 Fallback 处理。
    *   空状态（Empty State）的美化。
    *   加载中的骨架屏或 Loading 提示。

## 6. 预期效果

复刻完成后，医生端将呈现为一个**专业级**的医疗管理工具：
*   **视觉上**：对标行业头部 App，干净、现代。
*   **体验上**：操作路径短，信息获取快。
*   **功能上**：不仅能看数据，还能无缝沟通，实现“管理+服务”的闭环。
