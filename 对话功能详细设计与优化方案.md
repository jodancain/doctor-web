# 对话功能详细设计与优化方案

## 1. 竞品分析与差距

| 功能模块 | 竞品（微信/丁香医生） | 当前项目 | 差距与优化点 |
| :--- | :--- | :--- | :--- |
| **消息类型** | 支持文本、图片、语音、视频、卡片 | 仅支持文本 | **需增加多媒体消息支持**，至少支持图片发送。 |
| **状态反馈** | 发送中、发送失败重试、已读回执 | 仅有发送 loading，失败仅 Toast | **需优化发送状态 UI**，增加重发按钮；已读状态需后端支持。 |
| **实时性** | WebSocket 长连接，毫秒级延迟 | 轮询（5s/次），延迟高，耗流量 | 考虑引入 **WebSocket** 或优化轮询策略（如页面活跃时高频，后台低频）。 |
| **历史记录** | 下拉加载更多，本地持久化存储 | 一次性加载 100 条，无本地缓存 | **需实现分页加载**（上拉加载旧消息）和**本地缓存**以提升首屏速度。 |
| **UI/UX** | 气泡自适应、键盘避让流畅、头像点击 | 基础气泡，键盘遮挡偶发 | **优化输入框键盘交互**，增加表情面板入口（预留）。 |
| **消息操作** | 撤回、删除、复制、引用 | 无 | **增加长按菜单**，支持复制和删除单条消息。 |

## 2. 功能优化规划

### 2.1 UI/UX 交互升级
- **输入区域重构**：
  - 增加“更多”按钮（+），展开可发送图片/拍照。
  - 输入框高度自适应内容（多行文本）。
  - 解决 iOS 底部安全区 (`safe-area-inset-bottom`) 适配问题。
- **消息气泡优化**：
  - 优化气泡圆角和最大宽度，避免长文本溢出。
  - 增加发送时间的分组显示（如“5分钟前”），而非每条都显示时间。
- **状态反馈**：
  - 消息发送后先上屏（本地临时状态），待接口返回成功后更新状态。
  - 发送失败显示红色感叹号，点击可重试。

### 2.2 功能增强
- **图片发送**（已实现并优化）：
  - **选择方式优化**：
    - 采用操作菜单（相册/拍照），交互更清晰。
    - 支持多选图片（最多9张），参考主流聊天软件模式。
    - 使用 `chooseMedia` API（基础库 2.10.0+），向下兼容降级到 `chooseImage`。
  - **图片处理**：
    - 自动压缩大于2MB的图片（压缩质量80%），减少上传时间。
    - 图片大小检查（限制10MB），超出限制自动跳过并提示。
    - 上传前获取文件信息，提前验证文件大小。
  - **上传体验**：
    - 实时上传进度显示（0-100%），绿色进度条叠加在图片上。
    - 优化状态管理：`uploading` → `success`/`fail`。
    - 上传中图片半透明显示，视觉反馈清晰。
    - 改进错误提示，区分网络错误、文件过大等场景。
  - **预览和保存**：
    - 支持图片预览时左右滑动查看多张图片。
    - 长按图片显示操作菜单（保存图片/查看原图）。
    - 支持保存图片到相册，自动处理云存储 fileID 转换。
    - 保存失败时提示授权，引导用户开启相册权限。
  - **数据结构**：
    - 消息数据结构包含 `msgType: 'image'` 和 `content`（fileID）字段。
    - 前端添加 `tempUrl`（显示URL）、`thumbnailUrl`（缩略图）、`uploadProgress`（上传进度）等字段。
- **分页加载**：
  - 首次进入加载最近 20 条。
  - 下拉顶部触发 `scroll-view` 的 `bindrefresherrefresh`，加载更早的 20 条。
- **自动滚动**：
  - 发送新消息、收到新消息自动滚到底部。
  - 查看历史消息时保持滚动位置不变。

### 2.3 性能与架构优化
- **本地缓存策略**：
  - `onUnload` 时将当前会话的最新 N 条消息写入 `wx.setStorage`。
  - `onLoad` 时优先读取缓存渲染，再静默发起网络请求更新，消除白屏等待。
- **长轮询机制**（暂不引入 WS 的折中方案）：
  - 页面可见 (`onShow`) 时开启轮询，不可见 (`onHide`) 立即停止。
  - 轮询间隔动态调整：用户有操作时（如发送消息后）短期加密轮询（1s），无操作渐变至 5s。

  "_id": "msg_id",

## 4. 实施路线图
1.  **第一阶段（UI 调优）**：优化聊天气泡样式、时间显示策略、输入框键盘适配。
2.  **第二阶段（功能完善）**：实现图片发送、长按复制/删除、消息失败重试。
3.  **第三阶段（性能体验）**：引入本地缓存、下拉加载更多历史消息。
