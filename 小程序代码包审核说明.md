# 小程序代码包审核（代码质量）说明

本文说明与微信开发者工具 **代码质量 → 代码包** 相关的常见项，以及本项目的配置与排障方式。

## 1. 组件：启用组件按需注入（未通过时）

### 审核要求

- 需在微信小程序根配置中开启 **组件按需注入**。
- 对应 `app.json` 根级字段：

```json
"lazyCodeLoading": "requiredComponents"
```

### 本项目状态

- 已在 `miniprogram/app.json` 中配置上述项，用于通过「组件按需注入」检查，并减少未访问页面的无效注入。

### 与历史问题的关系

- 过去在 **Windows 微信开发者工具** 中，开启按需注入后，进入含 **Canvas 2D** 等较重能力的**分包页面**时，偶发 `navigateTo:fail timeout`。
- 当时曾通过**关闭** `lazyCodeLoading` 规避工具端问题，但会导致 **代码包审核该项不通过**。
- **当前做法**：保持 `lazyCodeLoading` **开启**，并对易超时分包使用 `preloadRule` 预下载（见下文），真机验证为主。

## 2. 预下载分包（缓解首次进入子包超时）

在 `miniprogram/app.json` 中配置：

```json
"preloadRule": {
  "pages/home/index": {
    "network": "all",
    "packages": ["packages/records"]
  }
}
```

含义：用户打开首页后，在 WiFi/移动网络下预下载 `packages/records` 分包，缩短首次进入「尿酸记录」「发作记录」等页面的等待时间。

可按需为其他分包增加条目（注意 `packages` 填写为 **分包 root 字符串**，与 `subpackages[].root` 一致）。

## 3. 其他常见代码包项

| 检查项 | 常见含义 | 建议 |
| :--- | :--- | :--- |
| 不建议引用过大插件（>200K） | 插件包体过大 | 精简插件或换方案 |
| 图片和音频资源 ≤200K | 静态资源过大 | 压缩图片、使用合适格式与尺寸 |

## 4. 仍出现 `navigateTo:fail timeout` 时

1. 使用 **真机预览/真机调试** 复测（工具与真机表现可能不一致）。
2. 升级 **微信开发者工具** 与 **基础库** 版本。
3. 确认目标页所在分包已被预下载或二次进入是否仍超时。
4. **上传发布前**请保持 `lazyCodeLoading: "requiredComponents"` 开启；勿为调试方便长期关闭，否则代码质量审核仍会失败。
