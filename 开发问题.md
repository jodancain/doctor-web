### 问题 #001: 病历夹功能点击打不开

**问题描述：**
- 用户从首页点击"病历夹"功能入口时，无法正常打开页面
- 页面跳转失败，没有响应

**问题分析：**

1. **tabBar 页面跳转问题：**
   - `pages/health-info/index` 是 tabBar 页面之一（配置在 `app.json` 的 `tabBar.list` 中）
   - 从首页使用 `wx.navigateTo()` 跳转到 tabBar 页面会导致失败
   - tabBar 页面应该使用 `wx.switchTab()` 进行跳转

2. **生命周期重复执行问题：**
   - `health-info/index.js` 中，`onLoad()` 和 `onShow()` 都调用了 `loadProfile()`
   - 如果用户未建立档案，两个生命周期都会触发 `wx.redirectTo()` 跳转到初始化页面
   - 这可能导致页面跳转冲突或重复执行

3. **延迟跳转问题：**
   - 使用 `setTimeout` 延迟 100ms 跳转，可能导致页面闪烁
   - 用户体验不佳，且可能在某些情况下导致跳转失败

**解决方案：**

1. **修复首页跳转逻辑：**
   - 在 `pages/home/index.js` 的 `handleFeatureTap()` 方法中
   - 判断目标页面是否为 tabBar 页面（`/pages/health-info/index`）
   - 如果是 tabBar 页面，使用 `wx.switchTab()` 跳转
   - 如果是普通页面，继续使用 `wx.navigateTo()` 跳转
   - 添加错误处理和用户提示

2. **优化病历夹页面生命周期：**
   - 在 `pages/health-info/index.js` 中分离逻辑：
     - `onLoad()`: 调用 `checkAndLoadProfile()`，只检查一次是否需要跳转到初始化页面
     - `onShow()`: 调用 `loadProfileData()`，只刷新数据，不进行跳转检查
   - 移除 `setTimeout` 延迟，直接使用 `wx.redirectTo()` 跳转
   - 添加错误处理和日志记录

3. **添加错误处理：**
   - 所有页面跳转操作添加 `fail` 回调
   - 跳转失败时显示用户友好的错误提示
   - 记录错误日志便于调试

**修改文件：**
- `miniprogram/pages/home/index.js` - 修复首页跳转逻辑
- `miniprogram/pages/health-info/index.js` - 优化生命周期和跳转逻辑

**测试要点：**
1. 从未建立档案的用户首次点击"病历夹"，应该跳转到初始化页面
2. 已建立档案的用户点击"病历夹"，应该正常显示十二宫格菜单
3. 从其他页面返回病历夹页面时，数据应该正确刷新
4. 点击菜单项进入详情页应该正常工作

**相关代码变更：**
- 详见 `CHANGELOG.md` 中的相关条目

**状态：** ✅ 已修复

- 用户从首页点击"病历夹"功能入口时，无法正常打开页面

   - 病历夹页面现在作为普通页面，统一使用 `navigateTo` 跳转。

2. **修复 `handlePendingTap` 函数：**
   - 将病历夹页面的跳转从 `switchTab` 改为 `navigateTo`。
   - 移除了错误的 `switchTab` 回退逻辑。

3. **明确 tabBar 页面列表：**
   - 只将真正在 `tabBar.list` 中注册的页面（首页、知识库、我的）使用 `switchTab`。
   - 其他所有页面都使用 `navigateTo`。

**修改文件：**

2. 从"待完善信息"点击"去评估"等按钮，应该能正确跳转到病历夹相关页面。
3. 不再出现 `switchTab:fail can not switch to no-tabBar page` 错误。

### 问题 #005: 患者端看不到知识库文章

**问题描述：**
- 患者端进入“健康知识库”页面后，显示“暂无相关文章”。
- 顶部“热门推荐”卡片仅显示标题栏，正文信息为空。
- 但医生端（管理后台）确认已存在已发布或历史文章数据。

**问题分析：**

1. **状态字段强匹配导致数据被误过滤：**
   - 列表查询固定使用 `status: '已发布'` 作为数据库查询条件。
   - 历史导入数据中，`status` 可能缺失，或使用了 `published`、`true`、`1` 等其他表达方式。
   - 结果是患者端把本应可见的文章全部过滤掉。

2. **列表页与详情页推荐逻辑不一致：**
   - 详情页相关推荐也固定使用 `status: '已发布'`，会出现列表可见但相关推荐缺失的潜在一致性问题。

3. **热门卡片缺少空数据保护：**
   - 页面未对 `hotArticle` 做条件渲染，极端情况下会显示“空壳卡片”。

**解决方案：**

1. **统一发布态兼容规则：**
   - 在 `miniprogram/pages/health-info/index.js` 与 `miniprogram/packages/resources/article/index.js` 增加 `isPublishedArticle` 方法。
   - 兼容状态：`已发布` / `发布` / `published` / `online` / `true` / `1` / 布尔 `true`，并兼容无 `status` 字段的历史数据。

2. **调整查询策略：**
   - 改为先读取 `education_articles` 再按兼容规则过滤，避免因单一状态值造成空列表。
   - 详情页相关推荐同样使用兼容过滤，并统一截取前 3 条结果。

3. **补齐空态渲染保护：**
   - `miniprogram/pages/health-info/index.wxml` 中为热门推荐区域增加 `wx:if="{{hotArticle}}"`。

**修改文件：**
- `miniprogram/pages/health-info/index.js`
- `miniprogram/pages/health-info/index.wxml`
- `miniprogram/packages/resources/article/index.js`

**测试要点：**
1. 数据库中仅有 `status: 'published'` 或 `status: true` 的文章时，患者端应可正常展示列表。
2. 数据库中 `status` 缺失的历史文章，患者端默认可见（除明确草稿外）。
3. 分类筛选后列表结果正确，详情页相关推荐不为空且不含草稿。
4. 当数据库完全无可见文章时，热门推荐区域不再出现空白卡片。

**相关代码变更：**
- 详见 `CHANGELOG.md` 中“修复患者端知识库文章‘全部为空’问题”条目。

### 问题 #006: 问卷详情页 JS 语法错误导致代码上传失败

**问题描述：**
- 在微信开发者工具中执行「上传」或编译时失败。
- 报错信息类似：`unknown: Expecting Unicode escape sequence \uXXXX. (55:21)`，文件为 `miniprogram/packages/records/questionnaire/detail/index.js`。

**问题分析：**
- `setData` 中用于按题目 `id` 写入 `answers` 的**对象键名**被破坏，出现非法反斜杠转义（如 `[\nswers.\\]` 等），不符合 JavaScript 语法。
- 可能由编辑器编码、错误合并或手动编辑时截断模板字符串导致。

**解决方案：**
- 将 `onSingleChange`、`onMultipleChange`、`onTextChange`、`onRatingChange` 中的 `setData` 统一写为（使用计算属性名按题目 id 更新 `answers`）：

```js
this.setData({ [`answers.${id}`]: e.detail.value });
```

- 与微信小程序 `setData` 支持的路径形式字段更新一致，保证按题目 id 写入 `answers` 子字段。

**修改文件：**
- `miniprogram/packages/records/questionnaire/detail/index.js`

**测试要点：**
1. 项目可正常编译，上传代码不再报 `Expecting Unicode escape sequence`。
2. 打开问卷详情页，单选/多选/文本/评分变更后，`answers` 中对应题目有值，提交校验通过。

**相关代码变更：**
- 详见 `CHANGELOG.md` 中「修复问卷详情页语法错误导致上传/编译失败」条目。

## 常见问题与解决方案

### Q1: tabBar 页面和普通页面的跳转有什么区别？
**A:** 
- **tabBar 页面**必须使用 `wx.switchTab()` 跳转
  - tabBar 页面必须在 `app.json` 的 `tabBar.list` 中注册
  - 当前项目中的 tabBar 页面：
    - `/pages/home/index`（首页）
    - `/pages/health-info/index`（知识库）
    - `/pages/account/index`（我的）
- **普通页面**使用 `wx.navigateTo()` 或 `wx.redirectTo()` 跳转
  - 普通页面在 `app.json` 的 `pages` 或 `subpackages` 中注册，但不在 `tabBar.list` 中
  - 例如：`/pages/medical-folder/index`（病历夹）、`/packages/records/ua-record/index`（尿酸记录）等
- **使用错误的方法会导致跳转失败**
  - 对普通页面使用 `switchTab` 会报错：`switchTab:fail can not switch to no-tabBar page`
  - 对 tabBar 页面使用 `navigateTo` 也可能失败或无法正确切换

### Q2: `onLoad` 和 `onShow` 有什么区别？
**A:**
- `onLoad`: 页面加载时执行一次，适合初始化逻辑
- `onShow`: 页面显示时执行（包括首次显示和从其他页面返回），适合刷新数据
- 避免在 `onShow` 中进行可能引起页面跳转的逻辑

### Q3: 如何避免重复跳转？
**A:**
- 在 `onLoad` 中检查并跳转
- 在 `onShow` 中只刷新数据，不跳转
- 使用标志位避免重复执行
