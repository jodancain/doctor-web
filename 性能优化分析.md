# 性能优化分析

## 1. 图片加载优化

### 现状分析
- **原图加载**: 聊天列表和详情页直接加载云存储的原图，导致大图（如相机拍摄的照片）加载缓慢，消耗大量流量。
- **未压缩上传**: 用户选择图片时未强制压缩，上传了数 MB 的高清图。

### 优化措施
1.  **客户端压缩上传**:
    -   在 `wx.chooseImage` 中设置 `sizeType: ['compressed']`，让微信客户端在上传前先进行有损压缩。
    -   预期收益：上传速度提升 3-5 倍，存储成本降低。

2.  **云端动态处理 (Image Processing)**:
    -   **列表缩略图**: 使用 `imageMogr2/thumbnail/400x/quality/80/interlace/1` 参数。
        -   `thumbnail/400x`: 限制宽度适配手机屏幕，大幅减小尺寸。
        -   `quality/80`: 降低质量至肉眼难辨差异，减少体积。
        -   `interlace/1`: 渐进式加载，提升弱网体验。
    -   **预期收益**: 列表加载速度提升 50% 以上，首屏渲染更快。

3.  **懒加载 (Lazy Load)**:
    -   在 `image` 组件上开启 `lazy-load`，仅当图片进入可视区域时才加载，减少首屏请求并发数。

## 2. 代码与包体积优化

### 现状分析
- 主包体积可能包含了非首屏必需的业务逻辑代码。
- 配置文件中部分编译选项未优化。

### 优化措施
1.  **分包策略**:
    -   将聊天、病历记录等非 TabBar 页面拆分至 `packages/consult`, `packages/records` 等分包。
    -   将仅分包使用的 JS 文件（如 `services/chat.js`）从主包移动到分包目录。
    -   预期收益：主包体积减小，小程序启动耗时（冷启动）降低。

2.  **按需注入**:
    -   `app.json` 中配置 `"lazyCodeLoading": "requiredComponents"`。
    -   预期收益：仅注入当前页面所需的自定义组件和逻辑，减少内存占用。

3.  **编译优化**:
    -   开启 `packNpmManually`，精细控制 NPM 包的构建。
    -   (尝试过) `useCompilerPlugins`，虽然目前因兼容性回退，但未来可继续跟进以利用更高级的编译压缩。

## 3. 渲染性能优化

### 现状分析
- 长列表（聊天记录）随着消息增多，DOM 节点数量激增，可能导致滚动卡顿。
- Canvas 动画可能导致页面卡顿和闪屏。
- Tab切换和数据加载时的渲染可能出现白屏。

### 优化措施
1.  **分页加载**: 目前已实现 `limit(20)` 的云函数查询，避免一次性拉取过多数据。
2.  **局部更新**: 使用 `setData` 时仅更新变更的数据字段，避免全量替换。
3.  **图片模式**: 使用 `aspectFill` 统一图片比例，避免因图片尺寸不一导致的重排（Reflow）。
4.  **硬件加速**:
    - 为需要动画的元素添加 `will-change` 属性，提示浏览器优化。
    - 使用 `transform: translateZ(0)` 强制开启硬件加速。
    - 适用于：卡片、列表项、进度条等动画元素。
5.  **Canvas 动画优化**:
    - 使用 `requestAnimationFrame` 确保动画与屏幕刷新率同步。
    - 优化 Canvas 绘制：预计算数据，减少动画循环中的计算。
    - 使用 `save/restore` 和一次性路径绘制，减少重绘次数。
    - 页面隐藏/卸载时停止动画，释放资源。
6.  **Tab切换优化**:
    - 缩短切换动画时长（0.25秒），使用 `ease-out` 缓动。
    - 延迟数据加载（100ms），避免切换动画卡顿。
    - 使用 `wx.nextTick` 确保 DOM 更新完成后再执行操作。
7.  **列表渲染优化**:
    - 使用唯一且稳定的 key（如 `createdAt`），避免重复渲染。
    - 嵌套循环时使用不同的索引变量名，避免作用域冲突。
8.  **加载状态管理**:
    - 添加加载状态标识，避免重复加载。
    - 数据为空时显示友好提示，避免白屏。
    - 错误处理时重置状态，避免界面卡住。

## 4. 动画性能优化

### 优化措施
1.  **缓动函数优化**:
    - 使用三次贝塞尔曲线（`cubic-bezier`）替代线性动画。
    - 饼状图使用平滑的缓动函数，避免突兀的动画效果。
2.  **动画时长优化**:
    - Tab切换：0.25秒（快速响应）。
    - 饼状图：0.8秒（视觉流畅）。
    - 进度条：0.6秒（平滑过渡）。
3.  **资源管理**:
    - 页面级变量存储动画ID和Canvas节点，避免重复创建。
    - 页面卸载时清理所有动画和资源，防止内存泄漏。
