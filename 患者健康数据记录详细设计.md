# 患者健康数据记录详细设计

## 1. 引言
本设计文档旨在详述"痛风/高尿酸血症慢病管理系统"中患者健康数据记录模块的详细设计。该模块是系统的核心功能之一，用于帮助患者记录和追踪关键健康指标，辅助医生进行病情评估和干预。

**注意**：本系统还包含**健康知识库**功能（`pages/health-info/index`），详细设计请参考 `知识库详细设计.md` 文档。病历夹功能已从 tabBar 移至首页入口，用户可通过首页"病历夹"功能访问。

## 2. 功能概述
系统支持以下几类健康数据的记录与管理：
1. **尿酸记录 (UA Records)**: 记录血尿酸值，监控病情控制情况。
2. **痛风发作记录 (Attack Records)**: 记录急性痛风发作的详情（部位、疼痛级、诱因等）。
3. **饮水打卡 (Water Records)**: 记录每日饮水量，促进排尿酸。
4. **运动记录 (Exercise Records)**: 记录运动类型和时长，辅助代谢管理。
5. **用药提醒与记录 (Medication Reminders)**: 设定服药提醒并记录服药情况。
6. **饮食记录 (Diet Records)**: 记录每日摄入食物及其嘌呤含量，支持按餐次分类和日期查看。

## 3. 数据库设计 (Database Schema)
所有集合均存储在云开发数据库中，通用字段包含 `_id`, `_openid`, `createdAt` (时间戳)。

### 3.1 尿酸记录 (`uaRecords`)
| 字段名 | 类型 | 描述 | 示例 |
| :--- | :--- | :--- | :--- |
| `value` | Number | 尿酸值 (μmol/L) | `420` |
| `measureDate` | Date/Number | 测量时间 | `1717234567890` |
| `type` | String | 测量类型 | `"hospital"` (医院), `"home"` (家用仪器) |
| `image` | String | (可选) 检查单照片FileID | `"cloud://..."` |
| `remark` | String | 备注 | `"空腹"` |

### 3.2 痛风发作记录 (`attackRecords`)
| 字段名 | 类型 | 描述 | 示例 |
| :--- | :--- | :--- | :--- |
| `startDate` | Date/Number | 发作开始时间 | `1717234567890` |
| `endDate` | Date/Number | (可选) 结束时间 | `1717234567890` |

### 3.3 饮水记录 (`waterRecords`)

### 3.4 运动记录 (`exerciseRecords`)

### 3.5 用药提醒/记录 (`medicationReminders`)
| 字段名 | 类型 | 描述 | 示例 |
| :--- | :--- | :--- | :--- |

### 3.6 饮食记录 (`dietRecords`)

## 4. 接口设计 (Cloud Functions)
统一使用 `healthDataService` 云函数，通过 `type` 参数区分操作。

### 4.1 新增/更新记录
*   **Action**: `addRecord` / `updateRecord` (需在 `services/record-service.js` 中补充 update 逻辑)
*   **Payload**:
    ```json
    {
      "type": "ua" | "attack" | "water" | "exercise" | "medication",
      "data": { ... } // 对应上述 schema 字段
    }
    ```
*   **Response**: `{ code: 0, data: { _id: "..." }, msg: "success" }`

### 4.2 获取记录列表
*   **Action**: `getRecords`
*   **Payload**:
    ```json
    {
      "type": "ua" | "attack" | "...",
      "limit": 20,
      "offset": 0,
      "startDate": "...", // 可选筛选
      "endDate": "..."    // 可选筛选
    }
    ```

### 4.3 获取首页/仪表盘摘要
*   **Action**: `getHomeSummary`
*   **Logic**: 并行查询各项最新数据，计算今日饮水总量、今日运动时长、最近一次尿酸值及日期、当前是否有发作记录。

## 5. 前端页面设计 (UI/UX)

### 5.1 通用交互原则
*   **快捷录入**: 首页提供“一键打卡”入口（特别是饮水和服药）。
*   **可视化反馈**: 录入完成后给予即时反馈（如弹窗、震动）。
*   **历史趋势**: 记录页应展示最近几次记录的列表或简易图表。

### 5.2 页面详细设计

#### 5.2.1 尿酸记录页 (`packages/records/ua-record`) [已重构]
*   **仪表盘设计 (Dashboard UI)**:
    *   **数值显示**: 顶部展示当前录入的尿酸值（超大号字体 `96rpx`，DIN 字体），配合状态标签（正常/偏高/危险）。
    *   **交互式滑动条**: 
        *   提供 `100-800` 范围的滑动条，用户可拖动快速调整数值。
        *   滑动条下方配有**三色分段条**（绿/黄/红），直观展示健康区间。
    *   **日期选择**: 采用卡片式布局的日期选择器，右侧箭头引导，操作清晰。
*   **趋势图表 (Canvas 2D Animation)**:
    *   **动画效果**: 实现了 `requestAnimationFrame` 驱动的生长动画，曲线从左向右平滑展开。
    *   **视觉优化**: 
        *   曲线采用贝塞尔插值 (`smoothness: 0.4`)，平滑顺畅。
        *   区域填充采用**垂直线性渐变**，增强立体感。
        *   自动绘制 `420` 警戒线（红色虚线），作为健康参考基准。
*   **数据交互**:
    *   实时监听滑动条变化，即时更新状态标签和颜色（绿色/黄色/红色）。
    *   点击保存后提供成功反馈并自动返回。
*   **历史记录列表 (`packages/records/ua-record/history`)**:
    *   **功能**: 展示所有历史尿酸记录，提供详细的查询视图。
    *   **UI 设计**:
      *   采用白色卡片容器，每条记录为独立的列表项。
      *   **左侧状态条**: 彩色竖线（绿色=正常，黄色=偏高），快速识别记录状态。
      *   **记录内容**: 
        *   左侧：日期（`30rpx` 加粗）、时间（`26rpx` 灰色）、备注标签（浅灰背景）。
        *   右侧：数值（`48rpx` 大号字体，颜色与状态对应）、单位、状态标签（圆角胶囊）。
      *   顶部显示记录总数统计。
    *   **交互**: 从记录页面点击"查看列表"链接跳转到历史记录页。

#### 5.2.2 痛风发作记录页 (`packages/records/attack-record`) [已实现并优化]
*   **Tab切换功能**:
    *   新增"记录发作"和"发作统计"两个标签页。
    *   支持在记录和统计视图之间快速切换。

    *   **疼痛程度**: 实现了滑动条交互（0-10级），联动显示动态表情（微笑/平淡/哭泣）和颜色（绿/黄/红）。
    *   **诱因记录**: 多选标签支持（饮酒、海鲜、受凉、高嘌呤饮食、熬夜劳累、剧烈运动）。
*   **数据提交**: 支持将发作详情提交至后端云函数。
*   **统计视图 (发作统计)**:
    *   **摘要统计卡片**:
        *   累计发作次数：显示总发作次数。
        *   最高疼痛等级：显示最高疼痛等级及对应的疼痛程度标签。
    *   **诱因分析（饼状图）**:
        *   使用Canvas 2D绘制环形饼状图（donut chart），展示不同诱因的占比。
        *   **动画效果**：实现了0.8秒的生长动画，使用平滑的三次贝塞尔缓动函数，饼状图从0%平滑展开到100%。

            *   使用`save/restore`和一次性路径绘制，减少Canvas重绘次数。
            *   预计算扇形角度，避免动画循环中重复计算。
            *   页面隐藏/卸载时自动停止动画，释放资源。
        *   支持最多5种颜色（红、橙、绿、蓝、紫），自动匹配诱因数据。
        *   图例显示：饼状图下方显示颜色与诱因名称的对应关系。
        *   自动统计各诱因出现的次数和占比。
    *   **高频发作部位**:
        *   显示前5个高频发作部位，按次数降序排列。
        *   每个部位显示序号（红色圆形）、名称、进度条（红色渐变）和次数。
        *   进度条采用动画效果（0.6秒，cubic-bezier缓动），宽度根据占比动态显示。
        *   使用硬件加速（`will-change: width`），确保动画流畅。
    *   **发作历史列表**:
        *   按时间倒序展示所有发作记录。
        *   每条记录包含：日期（YYYY-MM-DD格式）、发作部位、疼痛等级（彩色标签）、诱因标签。
        *   疼痛等级标签颜色：≥7级红色，≥4级黄色，<4级绿色。
        *   记录卡片左侧红色边框，增强视觉识别。
    *   **数据统计逻辑**:
        *   从云函数获取所有发作记录（`getRecords`）。
        *   自动计算累计次数、最高疼痛、诱因统计、部位统计。
        *   支持诱因和部位名称的标准化处理，确保统计准确性。

## 6. 数据安全与隐私
*   **权限控制**: 所有云数据库查询必须带 `_openid` 过滤，确保用户只能访问自己的数据。
*   **敏感数据**: 医疗数据属于敏感信息，建议在数据库侧开启安全规则（Security Rules），仅允许创建者读写。
*   **脱敏**: 如需进行全平台数据分析，必须去除身份信息。

## 7. 开发计划与优先级
1.  **P0 (核心)**: 尿酸记录、痛风发作记录。这是病程管理的基础。
2.  **P1 (高频)**: 饮水打卡、用药提醒。提高用户粘性。
3.  **P2 (辅助)**: 运动记录、饮食库对接。
4.  **P3 (高级)**: OCR 识别化验单、智能病情分析报告。
