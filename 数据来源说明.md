# 数据来源说明

## 概述

本系统的所有健康数据均从云数据库读取，已完全移除所有假数据和模拟数据。

## 数据来源

### 1. 健康记录数据

所有健康记录数据均从云数据库的以下集合读取：

- **尿酸记录** (`uaRecords`): 通过 `getRecords` 接口，`type: 'ua'` 获取
- **发作记录** (`attackRecords`): 通过 `getRecords` 接口，`type: 'attack'` 获取
- **饮水记录** (`waterRecords`): 通过 `getRecords` 接口，`type: 'water'` 获取
- **运动记录** (`exerciseRecords`): 通过 `getRecords` 接口，`type: 'exercise'` 获取
- **用药提醒** (`medicationReminders`): 通过 `getRecords` 接口，`type: 'medication'` 获取
- **饮食记录** (`dietRecords`): 通过 `getRecords` 接口，`type: 'diet'` 获取

### 2. 统计数据

统计数据通过以下接口从数据库实时计算：

- **首页汇总** (`getHomeSummary`): 统计最近尿酸值、近7日发作/饮水/运动/用药数据
- **数据看板** (`getDashboardData`): 提供7日详细统计、饮食分布、时间线等

### 3. 用户数据

用户数据从 `users` 集合读取，包括：
- 用户基本信息
- 医生患者绑定关系
- 登录状态等

## 数据调用方式

### 前端调用示例

#### 获取记录

```javascript
const { call } = require('../../utils/request');

// 获取用药提醒记录
const result = await call('getRecords', {
  type: 'medication',
  limit: 100
});

// 兼容处理：如果result是数组，直接使用；如果是对象且有data字段，取data
// 因为 request.js 中的 call 函数已经返回了 result.data
const records = Array.isArray(result) ? result : ((result && result.data) ? result.data : []);
```

#### 保存记录（以尿酸记录为例）

```javascript
const { call } = require('../../utils/request');

// 保存尿酸记录
const result = await call('addRecord', {
  type: 'ua',
  data: {
    value: 420,              // 尿酸值（必填）
    timestamp: Date.now(),    // 时间戳（必填）
    remark: '空腹'           // 备注（可选）
  }
});

if (result) {
  // 保存成功
  wx.showToast({ title: '记录成功', icon: 'success' });
} else {
  // 保存失败
  wx.showToast({ title: '保存失败', icon: 'none' });
}
```

### 云函数接口

所有数据通过 `healthDataService` 云函数获取：

- **接口名称**: `getRecords`
- **参数**: 
  - `type`: 记录类型（'ua'、'attack'、'water'、'exercise'、'medication'、'diet'）
  - `limit`: 返回记录数量限制（可选）
  - `offset`: 偏移量（可选）

## 数据隔离

所有数据查询自动按 `_openid` 字段进行隔离，确保：
- 用户只能访问自己的数据
- 数据安全性得到保障
- 符合隐私保护要求

## 注意事项

1. **数据为空时的处理**: 当数据库中没有数据时，页面会显示空状态，不会显示假数据
2. **加载失败处理**: 如果数据加载失败，会显示错误提示，不会使用模拟数据
3. **数据实时性**: 所有数据都是实时从数据库读取，确保数据的准确性和时效性

## 医生端患者管理接入说明

1. **工作台与待办事项**
   - 首页总患者数、高风险预警和近期活跃人数，通过 `getDoctorDashboardStats` 云函数动态读取真实注册用户（role: 'user'）计算。
   - 工作台下方的“待办事项”从真实患者列表中随机生成，并支持点击“处理”智能跳转至**AI对话界面**或**患者详情页**。

2. **患者管理列表**
   - 接入 `getDoctorPatients` 云函数获取数据库中真实的患者账号列表。
   - 配合生成了100名带有随机姓名的患者 `mock_patients_100_lines.json` 并已通过 JSON Lines 格式或 CSV 格式成功导入。

3. **患者详情页**
   - 通过 `getDoctorPatientOverview` 云函数获取患者真实姓名并计算相应的并发症风险和指标。
   - **查看健康档案**：支持点击后直接跳转至该名患者专属的 `/pages/medical-folder/index` 页面，展示健康病历夹。
   - **新建病历**：支持唤起 ActionSheet 选择（记录门诊病历、记录随访记录、上传检查报告），并能带上患者 ID 跳转至健康信息记录表单 `/pages/health-info/detail`。
