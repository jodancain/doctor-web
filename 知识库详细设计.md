# 健康知识库详细设计

## 1. 功能概述

健康知识库是痛风/高尿酸血症慢病管理系统中的核心内容模块，为用户提供专业的健康知识、饮食指导、用药指南和生活方式建议。该模块帮助患者更好地理解和管理疾病，提升自我管理能力。

## 2. 页面结构 (`pages/health-info/index`)

### 2.1 页面布局

页面采用垂直滚动布局，包含以下主要区域：

1. **分类筛选**：横向滚动的分类标签栏（页面顶部）
2. **热门推荐**：大卡片展示推荐文章
3. **最新文章列表**：文章卡片列表

**整体间距规范**：
- 页面内边距：顶部 `32rpx`，左右 `32rpx`，底部 `80rpx`
- 区域间距：分类筛选底部 `32rpx`，热门推荐底部 `56rpx`，文章区域顶部 `56rpx`
- 区域标题：字体 `36rpx`，加粗 (`font-weight: 700`)，底部间距 `32rpx`

**注意**：搜索功能已移除，页面聚焦于分类浏览和文章展示。

### 2.2 UI 组件设计

#### 2.2.1 分类筛选
- **分类**：
  - 全部（默认选中）
  - 饮食
  - 药物
  - 基础
  - 生活
- **样式**：
  - 未选中：白色背景，灰色文字 (`28rpx`, `font-weight: 500`)
  - 选中：青绿色背景 (`#0f766e`)，白色文字 (`font-weight: 600`)
- **尺寸**：内边距 `16rpx 36rpx`，最小高度 `64rpx`，圆角 `32rpx`
- **布局**：横向滚动，胶囊形状按钮，标签间距 `20rpx`

#### 2.2.2 热门推荐卡片
- **样式**：
  - 大卡片，青绿色渐变背景 (`#0f766e` -> `#14b8a6`)
  - 圆角 `28rpx`，内边距 `48rpx 44rpx`，带阴影
  - 白色文字，层次分明
- **内容**：
  - 顶部：标签"热门推荐" (`26rpx`, `font-weight: 500`) + 阅读时长（时钟图标 + 分钟数，`26rpx`）
  - 中间：文章标题（`40rpx` 加粗，行高 `1.5`，字间距 `0.5rpx`）
  - 底部：文章描述（`28rpx`，行高 `1.7`，半透明白色）
- **间距**：顶部间距 `28rpx`，标题底部间距 `20rpx`
- **交互**：点击跳转到文章详情页

#### 2.2.3 文章列表
- **布局**：垂直列表，每个文章为独立卡片，卡片间距 `28rpx`
- **卡片内容**：
  - 标题：`32rpx` 加粗，深灰色，行高 `1.6`，字间距 `0.3rpx`
  - 底部信息栏：
    - 左侧：分类标签（青绿色胶囊，白色文字，`24rpx`，内边距 `6rpx 20rpx`）
    - 右侧：阅读量（眼睛图标 + 数字）+ 阅读时长（`26rpx`）
- **样式**：
  - 白色背景，圆角 `28rpx`
  - 内边距：`40rpx 36rpx`
  - 柔和的阴影效果
  - 标题区域底部间距 `24rpx`，底部信息栏顶部间距 `8rpx`

### 2.3 数据模型

```javascript

  id: String,           // 文章ID

```

```javascript
{
  id: String,      // 分类ID

```

## 3. 功能逻辑

### 3.1 分类筛选
- **默认状态**：显示"全部"分类，展示所有文章
- **筛选逻辑**：点击分类后，只显示该分类下的文章

### 3.2 文章展示
- **热门推荐**：始终显示置顶的热门文章
- **列表排序**：按发布时间倒序（最新在前）
- **空状态**：筛选无结果时显示"暂无相关文章"

## 5. 云函数接口设计

### 5.1 获取文章列表
- **Action**: `getArticles`
- **Payload**:
  ```json
  {
    "category": "diet" | "medication" | "basic" | "lifestyle" | null,
    "limit": 20,
    "offset": 0
  }
  ```
- **Response**:
  ```json
  {
    "code": 0,
    "data": {
      "articles": [...],
      "total": 100
    }
  }
  ```

**注意**：搜索功能已移除，不再支持 `keyword` 参数。

### 5.2 获取热门文章
- **Action**: `getHotArticle`
- **Response**: 返回单个热门文章对象

### 5.3 获取文章详情
- **Action**: `getArticleDetail`
- **Payload**: `{ "articleId": "..." }`
- **Response**: 返回完整文章对象（包含 content）

### 5.4 更新阅读量
- **Action**: `incrementArticleViews`
- **Payload**: `{ "articleId": "..." }`
- **功能**: 文章被查看时，阅读量 +1

## 6. 交互流程

### 6.1 页面加载流程
1. 页面 `onLoad` 时调用 `getHotArticle` 获取热门文章
2. 调用 `getArticles` 获取文章列表（默认"全部"分类）
3. 调用 `filterArticles` 方法初始化文章列表
4. 渲染页面内容

### 6.2 分类筛选流程
1. 用户点击分类标签
2. 更新 `activeCategory` 状态
3. 调用 `filterArticles` 方法筛选
4. 更新列表显示

### 6.3 查看文章流程
1. 用户点击文章卡片
2. 调用 `incrementArticleViews` 增加阅读量
3. 跳转到文章详情页（待开发）
4. 或显示文章内容弹窗

## 8. 注意事项

1. **数据缓存**：文章列表可适当缓存，减少云函数调用
2. **性能优化**：大量文章时使用分页加载
3. **内容审核**：文章内容需由管理员审核后发布
4. **SEO 优化**：文章详情页支持分享到微信，优化分享卡片展示
