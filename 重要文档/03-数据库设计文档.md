# 03 — 数据库设计文档

## 一、概述

- **数据库**：MongoDB 6.0
- **连接方式**：`mongodb://admin:<password>@goutcare-mongo:27017/goutcare?authSource=admin`
- **初始化脚本**：`mongo-init.js`（Docker 首次启动时自动执行）
- **驱动**：MongoDB Node.js Driver 7.1（非 Mongoose，无 Schema 强制约束）

---

## 二、集合（Collections）总览

| 集合名 | 用途 | 主要索引 | 数据量级预估 |
|--------|------|---------|-------------|
| `users` | 医生/患者/管理员账户 | username(唯一), openid, role | 百级 |
| `uaRecords` | 尿酸值记录 | _openid + timestamp | 万级 |
| `attackRecords` | 痛风发作记录 | _openid + timestamp | 千级 |
| `waterRecords` | 饮水记录 | _openid + timestamp | 万级 |
| `exerciseRecords` | 运动记录 | _openid + timestamp | 万级 |
| `medicationReminders` | 用药提醒 | _openid + status | 千级 |
| `dietRecords` | 饮食记录 | _openid + timestamp | 万级 |
| `messages` | 医患聊天消息 | participants, clientMsgId(唯一) | 万级 |
| `education_articles` | 科普文章 | category + createdAt | 百级 |
| `questionnaires` | 问卷模板 | status | 十级 |
| `questionnaire_responses` | 问卷回答 | questionnaireId + respondent | 千级 |
| `projects` | 科研项目 | status | 十级 |
| `apiKeys` | Open API 密钥 | key(唯一) | 十级 |

```javascript

  department: "风湿免疫科",
  title: "主治医师",
  org: "XX医院/风湿免疫科",

  // --- 患者健康档案 ---
```

## 四、数据隔离策略

### 患者数据隔离
- 所有患者健康数据通过 `_openid` 字段隔离
- 小程序端请求必须携带 JWT，后端从 token 中提取 openid
- 查询时强制注入 `_openid` 条件，防止越权访问

### 医生数据权限
- 医生可通过 `/api/patients` 查看**所有**患者数据
- 未来可扩展为"我的患者"绑定关系

## 六、CloudBase 兼容层说明

`server/db.ts` 提供了一个模拟微信云开发 API 的适配层：

```typescript
// 云开发风格（小程序服务端代码依赖此接口）
db.collection('uaRecords')
  .where({ _openid: 'xxx' })
  .orderBy('timestamp', 'desc')
  .limit(20)
  .get()

// 实际底层转换为 MongoDB 原生操作
collection.find(filter).sort(sort).limit(limit).toArray()
```

**注意事项**：
- `update()` 方法**不**需要 `{ data: {...} }` 包裹（云开发需要，MongoDB 不需要）
- `remove()` 返回 `{ deleted: n }` 而非 `{ stats: { removed: n } }`
- `update()` 返回 `{ updated: n }` 而非 `{ stats: { updated: n } }`
- `_.inc(1)` 等操作符需替换为 MongoDB 原生 `{ $inc: { field: 1 } }`
