## 三、首次部署（从零开始）

### 3.1 一键部署脚本

```bash
# 1. SSH 登录服务器
ssh root@101.43.123.29

# 2. 上传项目文件到服务器
# （可用 scp、rsync 或通过 Git clone）
scp -r ./wechatcode-master root@101.43.123.29:/opt/goutcare

# 3. 运行部署脚本
cd /opt/goutcare
chmod +x deploy.sh
./deploy.sh
```

`deploy.sh` 会自动执行：
1. 安装 Docker + Docker Compose
2. 创建项目目录
3. 生成随机 `MONGO_PASSWORD` 和 `JWT_SECRET`
4. 申请 Let's Encrypt SSL 证书
5. 启动所有 Docker 容器

### 3.2 手动部署（逐步）

```bash
# 1. 安装 Docker
curl -fsSL https://get.docker.com | sh
systemctl enable docker && systemctl start docker

# 2. 进入项目目录
cd /opt/goutcare

# 3. 创建 .env 文件
cp .env.example .env
nano .env   # 编辑填写所有必需变量

# 4. 构建并启动
docker compose up -d --build

# 5. 查看日志
docker compose logs -f

# 6. 验证
curl http://localhost:3000/api/health
```

### 3.3 环境变量配置

创建 `/opt/goutcare/.env`：


## 四、Docker 容器管理

### 4.1 容器列表

| 容器名 | 镜像 | 端口 | 说明 |
|--------|------|------|------|
| `goutcare-backend` | 自建 Node.js | 3000 | Express 后端 |
| `goutcare-mongo` | mongo:6.0 | 27017(内部) | 数据库 |
| `goutcare-nginx` | nginx:latest | 80, 443 | 反向代理 |
| `goutcare-certbot` | certbot | - | SSL 证书续期 |

### 4.2 常用命令

```bash
# 查看所有容器状态
docker compose ps

# 查看日志（跟踪模式）
docker compose logs -f backend
docker compose logs -f nginx
docker compose logs -f mongo

# 重启单个服务
docker compose restart backend

# 重新构建并启动（代码更新后）
docker compose up -d --build backend

# 停止所有服务
docker compose down

# 停止并清除数据（危险！）
docker compose down -v
```

### 4.3 更新部署流程

```bash
# 1. 上传新代码到服务器
rsync -avz --exclude node_modules --exclude .env \
  ./wechatcode-master/ root@101.43.123.29:/opt/goutcare/

# 2. 重新构建并启动
ssh root@101.43.123.29 "cd /opt/goutcare && docker compose up -d --build backend"

# 3. 验证
curl https://yundongyl.cn/api/health
```

## 五、SSL 证书管理

### 自动续期
Certbot 容器每 12 小时检查并自动续期（证书有效期 90 天）。

### 手动续期
```bash
docker compose run --rm certbot renew
docker compose restart nginx
```

### 首次申请证书
```bash
docker compose run --rm certbot certonly \
  --webroot --webroot-path=/var/www/certbot \
  -d yundongyl.cn --email your@email.com --agree-tos
```

## 六、数据库运维

### 6.1 连接 MongoDB

```bash
# 进入容器
docker exec -it goutcare-mongo mongosh \
  "mongodb://admin:密码@localhost:27017/goutcare?authSource=admin"

# 常用查询
db.users.countDocuments({ role: 'patient' })    // 患者数
db.users.countDocuments({ role: 'doctor' })     // 医生数
db.uaRecords.countDocuments()                    // 尿酸记录数
db.messages.countDocuments()                     // 消息数
```

### 6.2 数据备份

```bash
# 手动备份
docker exec goutcare-mongo mongodump \
  --uri="mongodb://admin:密码@localhost:27017/goutcare?authSource=admin" \
  --out=/tmp/backup_$(date +%Y%m%d)

# 从容器复制到宿主机
docker cp goutcare-mongo:/tmp/backup_20260328 /opt/goutcare/backups/

# 建议：设置 crontab 每日自动备份
echo "0 3 * * * docker exec goutcare-mongo mongodump --uri='...' --out=/tmp/backup_\$(date +\%Y\%m\%d) && docker cp goutcare-mongo:/tmp/backup_\$(date +\%Y\%m\%d) /opt/goutcare/backups/" | crontab -
```

### 6.3 数据恢复

```bash
# 复制备份到容器
docker cp /opt/goutcare/backups/backup_20260328 goutcare-mongo:/tmp/

# 恢复
docker exec goutcare-mongo mongorestore \
  --uri="mongodb://admin:密码@localhost:27017/goutcare?authSource=admin" \
  --drop /tmp/backup_20260328/
```

## 七、Nginx 配置

配置文件：`nginx.conf`

### 关键配置点

```nginx
# HTTPS (443)
server {
    listen 443 ssl;
    server_name yundongyl.cn;

    ssl_certificate /etc/letsencrypt/live/yundongyl.cn/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/yundongyl.cn/privkey.pem;
    ssl_protocols TLSv1.2 TLSv1.3;

    # API 代理
    location /api/ {
        proxy_pass http://backend:3000;
        proxy_http_version 1.1;
        proxy_set_header Connection '';          # SSE 支持
        proxy_buffering off;                     # SSE 支持
        proxy_read_timeout 300s;                 # AI 长请求
    }

    # 静态上传文件
    location /uploads/ {
        alias /app/uploads/;
        expires 30d;
    }

    # SPA 前端
    location / {
        proxy_pass http://backend:3000;
    }
}

# HTTP → HTTPS 重定向
server {
    listen 80;
    location / { return 301 https://$host$request_uri; }
    location /.well-known/acme-challenge/ { root /var/www/certbot; }
}
```

## 九、故障处理

### 9.1 网站无法访问

```bash
# 1. 检查容器状态
docker compose ps

# 2. 如果容器挂了，查看日志
docker compose logs --tail=50 backend
docker compose logs --tail=50 nginx

# 3. 重启所有服务
docker compose restart

# 4. 如果还是不行，重新构建
docker compose down && docker compose up -d --build
```

### 9.2 SSL 证书过期

```bash
docker compose run --rm certbot renew --force-renewal
docker compose restart nginx
```

### 9.3 MongoDB 连接失败

```bash
# 检查 MongoDB 容器是否运行
docker compose ps mongo

# 查看 MongoDB 日志
docker compose logs mongo

# 检查磁盘空间
df -h

# 重启 MongoDB
docker compose restart mongo
```

### 9.4 磁盘空间不足

```bash
# 清理 Docker 未使用的资源
docker system prune -a

# 清理旧日志
docker compose logs --no-log-prefix backend | wc -l  # 查看日志量
truncate -s 0 $(docker inspect --format='{{.LogPath}}' goutcare-backend)
```
