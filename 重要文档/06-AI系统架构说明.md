# 06 — AI 系统架构说明

## 一、AI 功能概述

系统中有**两个 AI 对话入口**，底层都接入腾讯元器智能体"小风"：

| 入口 | 用户 | 模式 | 文件 |
|------|------|------|------|
| 小程序 AI 聊天 | 患者 | 非流式（一次返回完整回答） | `ai-service.cjs` |
| 医生端 AI 助手 | 医生 | SSE 流式（逐字返回） | `server/routes/ai.ts` |

---

## 二、腾讯元器 (Yuanqi) 集成

### 2.1 什么是腾讯元器
- 腾讯出品的 AI 智能体平台（类似 GPT Store）
- 支持自定义 System Prompt、知识库、工具调用
- API 兼容 OpenAI 格式，支持流式/非流式

### 2.2 智能体"小风"
- **平台**：yuanqi.tencent.com
- **APP_ID**：`kGvJbghrVRcy`
- **角色设定**：痛风/高尿酸血症健康管理助手
- **知识库**：在元器控制台配置（痛风指南、药物说明等）
- **System Prompt**：在元器控制台管理（不在代码中硬编码）

### 2.3 API 调用方式

**非流式（患者端）**：
```javascript
POST https://yuanqi.tencent.com/openapi/v1/agent/chat/completions
Headers:
  X-Source: openapi
  Authorization: Bearer ${YUANQI_APP_KEY}
  Content-Type: application/json

Body:
{
  "assistant_id": "${YUANQI_APP_ID}",
  "user_id": "${openid}",      // 用于会话隔离
  "stream": false,
  "messages": [
    { "role": "user", "content": "我尿酸420需要注意什么？" }
  ]
}
```

**流式（医生端）**：
```javascript
// 同上，但 stream: true
// 响应为 SSE 格式：
// data: {"choices":[{"delta":{"content":"片段文本"}}]}
// data: [DONE]
```

### 2.4 配置变量

| 变量 | 值 | 获取方式 |
|------|-----|---------|
| `YUANQI_APP_ID` | `kGvJbghrVRcy` | 元器控制台 → 智能体详情 |
| `YUANQI_APP_KEY` | （需获取） | 元器控制台 → 发布 → API接入 → 复制Token |

## 三、多 AI Provider 框架

### 3.1 架构设计

```
        请求
         │
         ▼
┌─────────────────────┐
│  AI Provider 注册表  │  ← ai-provider-registry.ts
│  (Registry)          │
└────────┬────────────┘
         │ 根据 AI_DEFAULT_PROVIDER 选择
         ▼
┌────────┼────────┬────────────┐
│        │        │            │
▼        ▼        ▼            ▼
Yuanqi  Claude   OpenAI    (可扩展)
Provider Provider Provider
│        │        │
▼        ▼        ▼
腾讯元器  Anthropic OpenAI
API      API      API
```

### 3.2 Provider 接口定义

```typescript
// ai-provider.ts
interface AIProvider {
  name: string;
  chat(messages: Message[], options?: ChatOptions): Promise<string>;
  streamChat(messages: Message[], options?: ChatOptions): AsyncGenerator<string>;
  isAvailable(): boolean;
}
```

### 3.3 已实现的 Provider

| Provider | 文件 | 环境变量 | 状态 |
|----------|------|---------|------|
| 腾讯元器 | `providers/yuanqi.ts` | `YUANQI_APP_ID` + `YUANQI_APP_KEY` | ✅ 主用 |
| Claude | `providers/claude.ts` | `CLAUDE_API_KEY` | ✅ 可选 |
| OpenAI | `providers/openai.ts` | `OPENAI_API_KEY` | ✅ 可选 |

### 3.4 切换 Provider

修改 `.env` 即可：
```env
AI_DEFAULT_PROVIDER=yuanqi   # 或 claude / openai
```

### 3.5 扩展新 Provider

```typescript
// 1. 创建 providers/new-provider.ts
import { AIProvider } from '../ai-provider';

export class NewProvider implements AIProvider {
  name = 'new-provider';
  async chat(messages, options) { /* ... */ }
  async *streamChat(messages, options) { /* ... */ }
  isAvailable() { return !!process.env.NEW_API_KEY; }
}

// 2. 在 ai-provider-registry.ts 中注册
registry.register(new NewProvider());
```

## 四、健康上下文注入

### 4.1 核心理念

AI 仅凭对话历史无法给出个性化建议。**健康上下文服务**在每次 AI 调用前，自动从数据库聚合患者的健康数据，注入到 system message 中。

### 4.2 数据采集器 (Collectors)

```
health-context.ts
  ├── profileCollector     → 基本信息（年龄、性别、BMI、并发症）
  ├── uaCollector          → 近期尿酸趋势（最近10条）
  ├── attackCollector      → 发作记录统计（频率、部位、诱因）
  ├── medicationCollector  → 当前用药方案
  ├── waterCollector       → 饮水量统计（7日均值）
  ├── exerciseCollector    → 运动情况统计
  ├── dietCollector        → 饮食记录摘要
  ├── questionnaireCollector → 问卷评估结果
  └── systemStatsCollector → 数据记录天数、活跃度
```

### 4.3 角色权限过滤

不同角色获取不同范围的数据：

| 角色 | 获取数据范围 |
|------|------------|
| `patient` | 自己的全部健康数据 |
| `doctor` | 指定患者的全部健康数据 |
| `admin` | 系统级统计数据 |
| `external` | 脱敏后的患者数据（Open API） |

### 4.4 上下文注入示例

AI 实际收到的 messages 结构：
```json
[
  {
    "role": "system",
    "content": "以下是该患者的健康数据摘要：\n基本信息：男，45岁，BMI 26.1\n尿酸趋势：近10次平均 385μmol/L，最高 520，最低 310\n发作记录：近3月发作2次，主要部位：左脚大拇指\n用药：非布司他 40mg 每日一次\n饮水：日均2100ml\n运动：每周3次，以散步为主\n..."
  },
  { "role": "user", "content": "我最近该怎么调整用药？" }
]
```

## 五、Open API（外部 AI 接入）

### 5.1 设计目的
- 允许外部更强大的 AI（如 Claude、GPT-4）读取患者数据
- 外部 AI 可以评估元器"小风"的回答质量
- 支持 A/B 测试不同 AI 的表现

### 5.2 接口总览

```
GET  /api/open-api/v1/patients              → 患者列表（脱敏）
GET  /api/open-api/v1/patients/:id/context  → 患者健康上下文
POST /api/open-api/v1/ai/test               → 测试 AI 回答
POST /api/open-api/v1/ai/evaluate           → 批量评估
POST /api/open-api/v1/ai/knowledge          → 上传知识
```

### 5.3 认证方式

```bash
# API Key 存储在 MongoDB apiKeys 集合
curl -H "Authorization: Bearer your-api-key" \
  https://yundongyl.cn/api/open-api/v1/patients
```

### 5.4 使用场景

**场景1：用 Claude 来评估元器回答**
```bash
# 1. 获取患者上下文
context=$(curl -s -H "Authorization: Bearer $KEY" \
  https://yundongyl.cn/api/open-api/v1/patients/xxx/context)

# 2. 让元器回答问题
answer=$(curl -s -X POST -H "Authorization: Bearer $KEY" \
  -d '{"question":"尿酸高怎么办？","patient_id":"xxx"}' \
  https://yundongyl.cn/api/open-api/v1/ai/test)

# 3. 把上下文+元器回答发给 Claude 评估
# （在 Claude/Cursor 中操作）
```

**场景2：A/B 测试多个 AI**
```bash
curl -X POST -H "Authorization: Bearer $KEY" \
  -d '{
    "questions": ["尿酸420怎么办", "非布司他副作用"],
    "patient_id": "xxx",
    "providers": ["yuanqi", "claude"]
  }' \
  https://yundongyl.cn/api/open-api/v1/ai/evaluate
```

## 七、元器控制台管理

### 7.1 修改 AI 人设
登录 yuanqi.tencent.com → 找到"小风" → 编辑 → 修改 System Prompt

### 7.2 管理知识库
元器控制台 → 知识库 → 上传文档（PDF/TXT）
- 推荐上传：痛风诊疗指南、药物说明书、食物嘌呤表

### 7.3 查看对话记录
元器控制台 → 数据统计 → 可查看所有 user_id 的对话历史

### 7.4 获取 API Key
元器控制台 → 发布 → API 接入 → 复制 Token → 填入 `.env` 的 `YUANQI_APP_KEY`
