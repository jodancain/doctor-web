# 07 — 安全机制说明

## 一、已实施的安全措施

### 1.1 认证与授权

| 机制 | 实现 | 文件 |
|------|------|------|
| JWT 认证 | jsonwebtoken 签发/验证 | `middleware/auth.ts` |
| 密码加密 | bcrypt (10轮 salt) | `user-service.cjs` |
| HttpOnly Cookie | 医生端 JWT 存储 | `auth.ts` |
| Bearer Token | 小程序端 JWT 传输 | `request.js` |
| API Key 认证 | Open API 独立认证 | `api-key-auth.ts` |
| 注册码保护 | 医生注册需验证码 | 环境变量 `DOCTOR_REGISTER_CODE` |

### 1.2 网络安全

| 机制 | 实现 |
|------|------|
| HTTPS 强制 | Nginx HTTP→HTTPS 301 重定向 |
| TLS 1.2+ | Nginx ssl_protocols 配置 |
| HSTS | Strict-Transport-Security header |
| X-Frame-Options | DENY（防点击劫持） |
| X-Content-Type-Options | nosniff（防 MIME 嗅探） |
| X-XSS-Protection | 1; mode=block |
| CORS 白名单 | 仅允许 yundongyl.cn |
| MongoDB 不对外 | 127.0.0.1:27017（Docker 内部网络） |

### 1.3 防攻击措施

| 机制 | 实现 | 配置 |
|------|------|------|
| 登录限流 | express-rate-limit | 15分钟/10次失败 |
| ReDoS 防护 | 搜索关键词 regex 转义 | `patients.ts`, `education.ts` |
| 文件上传限制 | 白名单后缀 + 大小限制 | `.jpg/.png/.gif/.webp`, 5MB |
| 分页限制 | pageSize 上限 100 | 所有列表接口 |
| 数据隔离 | _openid 强制注入 | 小程序所有查询 |
| 患者角色校验 | 小程序接口验证 role | `miniprogram.ts` |

## 二、已修复的安全漏洞

> 这些漏洞已在 2026-03-26 修复，但了解它们有助于理解代码中的安全设计。

### 2.1 【严重】JWT Secret 硬编码 fallback

**问题**：`miniprogram.ts` 中 JWT 验证使用 `process.env.JWT_SECRET || 'secret'`
**风险**：如果环境变量未配置，所有 JWT 都可以用 'secret' 伪造
**修复**：移除 `|| 'secret'`，启动时检查 JWT_SECRET 必须存在

### 2.2 【严重】医生注册码硬编码

**问题**：`user-service.cjs` 中注册码写死为 `'18736'`
**风险**：泄露源码即泄露注册码
**修复**：改为从 `process.env.DOCTOR_REGISTER_CODE` 读取

### 2.3 【高危】登录限流被注释

**问题**：`auth.ts` 中 rate-limit 代码被注释掉
**风险**：暴力破解密码
**修复**：取消注释，启用 15分钟窗口/10次限制

### 2.4 【高危】密码明文存储

**问题**：用户注册时密码直接存入 MongoDB
**风险**：数据库泄露即泄露所有密码
**修复**：注册时 bcrypt 加密，登录时 bcrypt.compare

### 2.5 【中危】ReDoS 注入

**问题**：患者搜索 `new RegExp(keyword)` 未转义
**风险**：恶意正则导致服务器 CPU 100%
**修复**：`keyword.replace(/[.*+?^${}()|[\]\\]/g, '\\$&')`

### 2.6 【中危】MONGO_URI 默认 fallback

**问题**：`db.ts` 连接串有默认值 `mongodb://localhost:27017/goutcare`
**风险**：生产环境误连无密码的本地数据库
**修复**：MONGO_URI 未配置时 `process.exit(1)`

## 三、安全配置检查清单

部署前/定期检查：

- [ ] `.env` 文件权限是否为 600（仅 root 可读）
- [ ] `JWT_SECRET` 是否为随机强密码（≥32字符）
- [ ] `MONGO_PASSWORD` 是否为随机强密码
- [ ] `WX_SECRET` 是否已填写
- [ ] `YUANQI_APP_KEY` 是否已填写
- [ ] MongoDB 端口是否仅绑定 127.0.0.1
- [ ] 防火墙是否仅开放 80/443
- [ ] SSL 证书是否有效
- [ ] Docker 容器是否以非 root 用户运行
- [ ] `.env` 文件未提交到 Git 仓库

## 五、敏感信息位置提醒

| 文件 | 敏感内容 | 注意事项 |
|------|---------|---------|
| `.env` | 所有密钥和密码 | **绝不提交到 Git** |
| `project.private.config.json` | 微信开发者工具配置 | 已在 .gitignore |
