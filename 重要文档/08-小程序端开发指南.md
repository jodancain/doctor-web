## 三、核心机制

### 3.1 请求封装 (`utils/request.js`)

**所有后端请求**都通过 `request.js` 统一发送：

```javascript
// 使用方式
const { request } = require('../../utils/request');

// 发送请求（自动带 JWT）
const res = await request({
  action: 'getUARecords',
  page: 1,
  pageSize: 20
});
```

**工作原理**：
1. 从本地存储读取 JWT token
2. POST 到 `https://yundongyl.cn/api/miniprogram`
3. 请求体中包含 `action` 字段指定操作
4. 自动处理 401（token 过期）→ 清除登录态 → 跳转登录页
5. 返回 `{ code, msg, data }`

**环境切换**：
```javascript
const BASE_URL = {
  develop: 'http://localhost:3000',
  trial: 'https://yundongyl.cn',
  release: 'https://yundongyl.cn'
};
```

### 3.2 认证流程 (`utils/auth.js`)

```
用户打开小程序
  │
  ├── 有本地 token → 验证 token → 有效 → 进入首页
  │                              └── 无效 → 跳转登录
  │
  └── 无 token → 跳转登录页
         │
         ├── wx.login() 获取 code
         ├── POST /api/miniprogram { action: 'login', code }
         ├── 后端用 code 换 openid
         ├── 签发 JWT（含 openid）
         └── 前端存储 token → 进入首页
```

### 3.3 数据服务层 (`services/`)

- `services/statistics.js` — 健康数据统计计算（7日尿酸均值、饮水总量等）
- `services/user.js` — 用户信息管理

### 3.4 格式化工具 (`utils/format.js`)

提供日期格式化、数值单位转换等通用功能。

## 四、开发规范

### 4.1 页面文件规范

每个页面包含 4 个文件：
```
page-name/
├── index.wxml    ← 模板（类 HTML）
├── index.wxss    ← 样式（类 CSS）
├── index.js      ← 逻辑（Page/Component 实例）
└── index.json    ← 配置（导航栏标题等）
```

### 4.2 数据绑定

```javascript
// index.js
Page({
  data: {
    records: [],
    loading: true
  },

  onLoad() {
    this.loadRecords();
  },

  async loadRecords() {
    this.setData({ loading: true });
    const res = await request({ action: 'getUARecords' });
    if (res.code === 0) {
      this.setData({ records: res.data.records, loading: false });
    }
  }
});
```

```xml
<!-- index.wxml -->
<view wx:if="{{loading}}">加载中...</view>
<view wx:for="{{records}}" wx:key="_id">
  <text>尿酸值: {{item.value}} μmol/L</text>
</view>
```

### 4.3 子包预加载

在 `app.json` 中配置：
```json
{
  "preloadRule": {
    "pages/home/index": {
      "network": "all",
      "packages": ["packages/records"]
    }
  }
}
```

## 五、关键页面说明

### 5.1 尿酸记录页 (`packages/records/ua-record/`)

- 支持手动输入尿酸值
- 图表展示趋势（Canvas 绑定 chartData，直接操作 Canvas 动画，不走 setData）

- 对话列表展示
- 发送消息 → 调用 `services/ai.js` → 后端请求元器 → 返回 AI 回答
- 消息状态：发送中 / 已接收 / 错误

### 5.3 医患聊天页 (`packages/consult/chat/`)

- 轮询机制获取新消息（`_pollingTimer` 存储在实例属性上，非 data）
- 发送文本/图片消息
- 已读/未读状态
- `clientMsgId` 去重防止重复发送

## 六、常见问题

### Q: 如何添加新页面？
1. 在 `miniprogram/pages/` 或 `packages/` 下创建新目录
2. 在 `app.json` 的 `pages` 或对应 `subpackages` 中注册路径
3. 如需新 action，在后端 `miniprogram.ts` 中添加路由

### Q: 如何调试后端请求？
1. 微信开发者工具 → 详情 → 本地设置 → 勾选"不校验合法域名"
2. `request.js` 中 develop 环境指向 `http://localhost:3000`
3. 查看 Console 中的请求日志

### Q: 构建报包体过大怎么办？
1. 图片压缩或改用网络图片
2. 将非首屏页面移入子包
3. 检查是否引入了不必要的库
