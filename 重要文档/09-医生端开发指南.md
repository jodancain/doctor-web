# 09 — 医生端开发指南

## 一、概述

医生端是一个**全栈应用**，前端 React SPA + 后端 Express API 打包在同一个项目中。

| 项目 | 值 |
|------|-----|
| 前端框架 | React 19 + TypeScript |
| 构建工具 | Vite 6 |
| CSS 框架 | Tailwind CSS 4 |
| 后端框架 | Express 5 + TypeScript |
| 运行方式 | `tsx server.ts`（开发） / `node dist/server.js`（生产） |
| 代码目录 | `goutcare---doctor-portal (1)/` |

## 三、开发模式

### 3.1 启动开发服务器

```bash
cd "goutcare---doctor-portal (1)"
cp .env.example .env
# 编辑 .env 填写必要变量（至少 MONGO_URI, JWT_SECRET）

npm install
npx tsx server.ts
```

开发模式下 Vite 中间件提供 HMR（热更新），前后端同时运行在 `:3000`。

### 3.2 构建生产版本

```bash
npx vite build          # 输出到 dist/
NODE_ENV=production npx tsx server.ts   # 生产模式启动
```

## 四、后端开发

### 4.1 路由注册

所有路由在 `server.ts` 中注册：

```typescript
// server.ts
import authRoutes from './server/routes/auth';
import patientRoutes from './server/routes/patients';
// ...

app.use('/api/auth', authRoutes);
app.use('/api/patients', patientRoutes);
// ...
```

### 4.2 添加新路由

```typescript
// server/routes/new-feature.ts
import { Router } from 'express';
import { requireAuth, AuthRequest } from '../middleware/auth';
import { getDb } from '../db';

const router = Router();

router.get('/', requireAuth, async (req: AuthRequest, res) => {
  const db = getDb();
  const items = await db.collection('new_collection').find({}).toArray();
  res.json({ success: true, data: items });
});

export default router;

// 然后在 server.ts 中注册：
// import newFeatureRoutes from './server/routes/new-feature';
// app.use('/api/new-feature', newFeatureRoutes);
```

### 4.3 数据库操作

**医生端路由** — 直接使用 MongoDB 原生 Driver：
```typescript
const db = getDb();
const collection = db.collection('uaRecords');

// 查询
const records = await collection.find({ _openid: patientId })
  .sort({ timestamp: -1 })
  .limit(20)
  .toArray();

// 插入
const result = await collection.insertOne({ ... });

// 更新
await collection.updateOne(
  { _id: new ObjectId(id) },
  { $set: { status: 'active' } }
);
```

**小程序服务** — 使用 CloudBase 兼容层（.cjs 文件）：
```javascript
const db = cloud.database();
const res = await db.collection('uaRecords')
  .where({ _openid: openid })
  .orderBy('timestamp', 'desc')
  .limit(20)
  .get();
```

> **注意**：两种方式底层都是同一个 MongoDB，但 API 风格不同。新功能建议用原生 Driver。

### 4.4 认证中间件

```typescript
import { requireAuth, AuthRequest } from '../middleware/auth';

// requireAuth 会：
// 1. 从 Cookie 或 Authorization header 中提取 JWT
// 2. 验证 JWT 签名
// 3. 将解码后的用户信息挂到 req.user
// 4. 失败返回 401

router.get('/protected', requireAuth, (req: AuthRequest, res) => {
  console.log(req.user); // { id, username, role, ... }
});
```

### 5.2 API 调用

前端直接使用 `fetch`，Cookie 自动携带：

```typescript
// 获取数据
const res = await fetch('/api/patients');
const data = await res.json();

// 提交数据
await fetch('/api/education', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ title, content, category })
});
```

### 5.3 SSE 流式 AI 对话

```typescript
// AIChat.tsx 中的核心逻辑
const res = await fetch('/api/ai/chat', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ message, history })
});

const reader = res.body.getReader();
const decoder = new TextDecoder();

while (true) {
  const { done, value } = await reader.read();
  if (done) break;

  const chunk = decoder.decode(value);
  // 解析 SSE: "data: {...}\n\n"
  const lines = chunk.split('\n');
  for (const line of lines) {
    if (line.startsWith('data: ') && line !== 'data: [DONE]') {
      const json = JSON.parse(line.slice(6));
      const text = json.choices[0]?.delta?.content || '';
      // 追加到消息显示
    }
  }
}
```

## 六、小程序后端服务（.cjs 文件）

### 为什么是 .cjs？
这些文件最初是微信云函数（CommonJS 模块），迁移到 Express 后保留了原始格式，通过 `miniprogram.ts` 桥接调用。

### 调用链路
```
miniprogram.ts 接收请求
  → 根据 action 字段选择 service
  → 调用对应 service 的函数
  → service 使用 CloudBase 兼容层操作数据库
  → 返回 { code, msg, data }
```

### 修改 service 时注意
1. **不需要** `{ data: {...} }` 包裹 — 直接传字段
2. 使用 MongoDB 原生操作符（`$set`, `$inc`）而非 CloudBase 操作符（`_.inc()`）
3. `update()` 返回 `{ updated: n }` 而非 `{ stats: { updated: n } }`
4. `remove()` 返回 `{ deleted: n }` 而非 `{ stats: { removed: n } }`

## 七、常见开发任务

### 添加新的数据记录类型

1. `mongo-init.js` — 添加新集合和索引
2. `record-service.cjs` — 添加 add/get/delete 方法
3. `miniprogram.ts` — 注册新 action
4. 小程序端 — 创建新页面，调用新 action
5. `patients.ts` — 医生端可查看该类型数据

### 添加新的医生端页面

1. `pages/` 下创建新的 `.tsx` 文件
2. `App.tsx` 中添加路由
3. 如需后端 API，在 `server/routes/` 添加路由文件
4. `server.ts` 中注册路由

### 修改 AI 行为

- **修改 AI 人设**：在腾讯元器控制台修改（不在代码中）
- **修改健康上下文**：编辑 `health-context.ts` 中的 Collector
- **切换 AI 供应商**：修改 `.env` 中的 `AI_DEFAULT_PROVIDER`
