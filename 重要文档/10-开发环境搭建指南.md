## 二、后端环境搭建

### 方式一：本地 MongoDB

```bash
# 1. 安装 MongoDB（macOS）
brew install mongodb-community
brew services start mongodb-community

# 1. 安装 MongoDB（Windows）
# 下载 MSI 安装包，安装时选择"Complete"，启动 MongoDB Service

# 2. 初始化数据库
mongosh < mongo-init.js

# 3. 进入项目目录
cd "goutcare---doctor-portal (1)"

# 4. 安装依赖
npm install

# 5. 配置环境变量
cp .env.example .env
```

编辑 `.env`：
```env
MONGO_URI=mongodb://localhost:27017/goutcare
JWT_SECRET=dev-secret-change-in-production
WX_APPID=wx88328469a06c102e
WX_SECRET=你的微信密钥（可先留空，登录功能不可用）
DOCTOR_REGISTER_CODE=18736
YUANQI_APP_ID=kGvJbghrVRcy
YUANQI_APP_KEY=你的元器密钥（可先留空，AI功能不可用）
PORT=3000
NODE_ENV=development
```

```bash
# 6. 启动服务
npx tsx server.ts
```

访问 http://localhost:3000 即可看到医生端页面。

### 方式二：Docker MongoDB

```bash
# 1. 启动 MongoDB 容器
docker run -d --name goutcare-mongo \
  -p 27017:27017 \
  -v goutcare_data:/data/db \
  mongo:6.0

# 2. 初始化数据库
docker exec -i goutcare-mongo mongosh < mongo-init.js

# 后续步骤同方式一，MONGO_URI 使用：
# mongodb://localhost:27017/goutcare
```

### 方式三：Docker Compose 全栈

```bash
# 一键启动所有服务（后端 + 数据库 + Nginx）
docker compose up -d --build

# 查看日志
docker compose logs -f backend
```

## 三、小程序开发环境

### 3.1 打开项目

1. 打开微信开发者工具
2. 选择"导入项目"
3. 项目目录选择 `wechatcode-master` 根目录
4. AppID 填写 `wx88328469a06c102e`（或使用测试号）
5. 确认导入

### 3.2 配置后端地址

编辑 `miniprogram/utils/request.js`：
```javascript
const BASE_URL = {
  develop: 'http://localhost:3000',   // ← 本地开发
  trial: 'https://yundongyl.cn',      // 体验版
  release: 'https://yundongyl.cn'     // 正式版
};
```

### 3.3 跳过域名校验

微信开发者工具 → 详情 → 本地设置 → ✅ 不校验合法域名、web-view（业务域名）、TLS 版本以及 HTTPS 证书

> 这样才能在开发模式下请求 localhost。

### 3.4 创建测试账号

后端启动后，在 MongoDB 中手动创建一个测试用户：

```javascript
// 在 mongosh 中执行
db.users.insertOne({
  _openid: "test-openid-001",
  nickName: "测试患者",
  role: "patient",
  createdAt: new Date().toISOString()
});
```

或者直接通过小程序登录流程（需要 WX_SECRET 配置正确）。

## 五、常见问题排查

### Q: `npm install` 报 bcrypt 编译错误
```bash
# Windows 需要安装 Visual Studio Build Tools
npm install --global windows-build-tools

# 或使用 bcryptjs 替代（纯 JS 实现，无需编译）
npm uninstall bcrypt && npm install bcryptjs
# 注意：需要修改代码中的 require('bcrypt') 为 require('bcryptjs')
```

### Q: MongoDB 连接失败
```bash
# 检查 MongoDB 是否运行
mongosh --eval "db.runCommand('ping')"

# 检查 .env 中的 MONGO_URI 是否正确
# 本地开发通常是：mongodb://localhost:27017/goutcare
```

### Q: 小程序请求 401
- 检查后端是否启动（http://localhost:3000/api/health）
- 检查 JWT_SECRET 是否配置
- 检查小程序本地存储中是否有有效 token
- 开发者工具 → 清除缓存 → 重新登录

### Q: AI 功能不可用
- 检查 `.env` 中 `YUANQI_APP_ID` 和 `YUANQI_APP_KEY` 是否填写
- AI 功能不影响其他功能使用，可先跳过

### Q: 端口 3000 被占用
```bash
# 查找占用端口的进程
lsof -i :3000   # macOS/Linux
netstat -ano | findstr :3000   # Windows

# 或修改 .env 中的 PORT
PORT=3001
```

## 六、开发工作流

```
1. 拉取最新代码
   git pull origin main

2. 安装依赖（如有新增）
   cd "goutcare---doctor-portal (1)" && npm install

3. 启动后端
   npx tsx server.ts

4. 打开微信开发者工具（小程序开发时）

5. 开发完成后测试
   - 医生端：浏览器访问 localhost:3000
   - 小程序：开发者工具中预览

6. 提交代码
   git add . && git commit -m "feat: xxx"
```
